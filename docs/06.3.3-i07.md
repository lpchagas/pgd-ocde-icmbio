# I07 — Horas por Entrega — Planejadas (Absolutas)

## 1. Finalidade do indicador

O I07 calcula o total de horas de trabalho planejadas alocadas a cada entrega do Plano de Entregas, somando a contribuição de todos os servidores cujos Planos de Trabalho se sobrepõem ao período analisado e que tenham a entrega vinculada.

O foco é em horas planejadas (não realizadas): o objetivo é diagnosticar o esforço previsto para cada entrega no horizonte do Plano de Entregas.

## 2. Abordagem técnica (versão Denodo/JDBC)

Observação importante: a versão canônica do script local (`Tabelas CSV/denodo_maio-2026/IND_07.1_run.py`) substitui a geração de calendário via `WITH RECURSIVE` (MySQL) por uma aproximação compatível com o driver Denodo VDP JDBC. Em vez de contar dias úteis excluindo feriados, a versão Denodo estima horas proporcionais a partir de `planos_trabalhos.carga_horaria` usando aritmética de datas disponível no Denodo.

Pontos principais:

- Não há CTE recursiva (compatibilidade Denodo VQL).
- A fórmula central é:

  horas_alocadas_entrega = carga_horaria_horas
                           × (overlap_dias / total_dias_plano)
                           × (forca_trabalho / 100)

  onde `carga_horaria_horas` é `pt.carga_horaria` (ou `pt.carga_horaria * 8` quando `forma_contagem_carga_horaria = 'DIAS'`).

- A aritmética de datas no Denodo usada é `(date1 - date2)` para diferença em dias (inclusive). Funções como `DIFF()`, `DATEDIFF()` ou `TIMESTAMPDIFF()` não estão disponíveis no driver; assim a expressão `(CAST(pt.data_fim AS DATE) - CAST(pt.data_inicio AS DATE)) + 1` é usada para contar dias inclusivos.

### SQL usado no script (CTE principal, adaptado do script canônico)

```sql
WITH parametros AS (
    SELECT
        CAST('{ini}' AS DATE) AS data_inicio,
        CAST('{fim}' AS DATE) AS data_fim,
        0                     AS incluir_excluidos
),
planos_horas AS (
    SELECT
        pt.id        AS plano_trabalho_id,
        pt.unidade_id,
        CASE pt.forma_contagem_carga_horaria
            WHEN 'DIAS' THEN pt.carga_horaria * 8.0
            ELSE             pt.carga_horaria
        END
        * (
            (CASE WHEN CAST(pt.data_fim   AS DATE) < p.data_fim
                  THEN CAST(pt.data_fim   AS DATE)
                  ELSE p.data_fim END)
            - (CASE WHEN CAST(pt.data_inicio AS DATE) > p.data_inicio
                    THEN CAST(pt.data_inicio AS DATE)
                    ELSE p.data_inicio END)
            + 1
          )
        / NULLIF(
              (CAST(pt.data_fim AS DATE) - CAST(pt.data_inicio AS DATE)) + 1,
              0
          )
        AS horas_proporcionais
    FROM petrvs_icmbio_planos_trabalhos pt
    CROSS JOIN parametros p
    WHERE CAST(pt.data_inicio AS DATE) <= p.data_fim
      AND CAST(pt.data_fim   AS DATE) >= p.data_inicio
      AND (p.incluir_excluidos = 1 OR pt.deleted_at IS NULL)
      AND pt.carga_horaria IS NOT NULL
      AND pt.carga_horaria > 0
),
vinculos_ativos AS (
    SELECT
        pte.plano_trabalho_id,
        pte.plano_entrega_entrega_id    AS id_entrega,
        COALESCE(pte.forca_trabalho, 0) AS forca_trabalho
    FROM petrvs_icmbio_planos_trabalhos_entregas pte
    WHERE pte.plano_entrega_entrega_id IS NOT NULL
      AND pte.deleted_at IS NULL
),
linhas AS (
    SELECT
        COALESCE(un.sigla, 'N.I.') AS unidade_sigla,
        COALESCE(un.nome,  'N.I.') AS unidade_nome,
        va.id_entrega,
        COALESCE(
            NULLIF(TRIM(COALESCE(pee.descricao,         '')), ''),
            NULLIF(TRIM(COALESCE(pee.descricao_entrega, '')), ''),
            'N.I.'
        )                          AS nome_entrega,
        pe.id                      AS id_plano_entrega,
        CAST(pe.data_inicio AS DATE) AS inicio_vigencia_plano_entrega,
        CAST(pe.data_fim    AS DATE) AS fim_vigencia_plano_entrega,
        ph.plano_trabalho_id,
        ph.horas_proporcionais * (va.forca_trabalho / 100.0) AS horas_servidor
    FROM vinculos_ativos va
    JOIN planos_horas ph
        ON ph.plano_trabalho_id = va.plano_trabalho_id
    LEFT JOIN petrvs_icmbio_planos_entregas_entregas pee
        ON pee.id = va.id_entrega
       AND pee.deleted_at IS NULL
    LEFT JOIN petrvs_icmbio_planos_entregas pe
        ON pe.id = pee.plano_entrega_id
       AND pe.deleted_at IS NULL
    LEFT JOIN petrvs_icmbio_unidades un
        ON un.id = ph.unidade_id
)
SELECT
    unidade_sigla,
    unidade_nome,
    id_entrega,
    nome_entrega,
    id_plano_entrega,
    inicio_vigencia_plano_entrega,
    fim_vigencia_plano_entrega,
    ROUND(SUM(horas_servidor), 2)         AS total_horas_planejadas_entrega,
    COUNT(DISTINCT plano_trabalho_id)     AS num_servidores_alocados
FROM linhas
GROUP BY
    unidade_sigla, unidade_nome, id_entrega, nome_entrega,
    id_plano_entrega, inicio_vigencia_plano_entrega, fim_vigencia_plano_entrega
ORDER BY unidade_sigla, total_horas_planejadas_entrega DESC;
```

## 4. Fluxo de execução detalhado (passo a passo)

1. Preparação
     - Verificar que `denodo_config` (ou variáveis de ambiente) estejam definidas: `JDBC_URL`, `DENODO_USER` e `DENODO_PASSWORD` (não versionar). O script canônico usa placeholders; **nunca** copie credenciais para `docs/`.
     - Validar existência do driver: `denodo-vdp-jdbcdriver.jar` e `jvm.dll` do JRE utilizado pelo DBeaver/ambiente.
     - Ajustar `build_periods()` se necessário (regra padrão: 2025 trimestral, 2026+ quadrimestral).

2. Rotina principal (executada pelo `IND_07.1_run.py`)
     - `start_jvm()` inicia a JVM via `jpype` (idempotente).
     - `get_conn()` cria `java.sql.Connection` usando `DriverManager` e `Properties(user,password)`.
     - Para cada período retornado por `build_periods()`:
         a. Formatar `SQL_I07.format(ini=..., fim=...)` e executar `stmt.executeQuery(sql)`.
         b. Ler `ResultSet`, extrair `ResultSetMetaData` e aplicar `clean()` em todos os campos (remove quebras de linha internas, converte `None` → `''`).
         c. Acumular linhas junto com as 6 colunas de metadado (`ciclo_tipo`, `periodo`, `periodo_inicio`, `periodo_fim`, `periodo_status`, `duracao_dias`).
         d. Em caso de erro SQL em um período, o script registra o erro e continua com os próximos períodos (try/except por período).
     - Ao final, se houver linhas, gerar `IND_07.2_horas_por_entrega_<stamp>.csv` no diretório `Tabelas CSV/denodo_maio-2026/` com `delimiter='|'` e `encoding='utf-8-sig'`.

3. Saída e logs
     - O script imprime por período: `Executando T1-2025 (YYYY-MM-DD a YYYY-MM-DD)... X linhas retornadas.`
     - Após salvar o CSV, o script imprime o total de linhas e o caminho do arquivo.

## 5. Tabelas e campos críticos (detalhado)

- `petrvs_icmbio_planos_trabalhos` (alias `pt`)
    - `id`, `unidade_id`, `carga_horaria`, `forma_contagem_carga_horaria`, `data_inicio`, `data_fim`, `deleted_at`
- `petrvs_icmbio_planos_trabalhos_entregas` (alias `pte`)

- `petrvs_icmbio_planos_entregas_entregas` (alias `pee`)
    - `id`, `plano_entrega_id`, `descricao`, `descricao_entrega`, `deleted_at`
- `petrvs_icmbio_planos_entregas` (alias `pe`)
    - `id`, `data_inicio`, `data_fim`, `status`, `deleted_at`
- `petrvs_icmbio_unidades` (alias `un`)
    - `id`, `sigla`, `nome`

Observações:
- Sempre aplicar `deleted_at IS NULL` nos JOINs quando apropriado para evitar registros soft-deleted.
- `forma_contagem_carga_horaria = 'DIAS'` exige multiplicação por 8.0 para obter horas.

## 6. Dicionário de saída completo (colunas do CSV + tipos esperados)

- `ciclo_tipo` (string): `trimestral` | `quadrimestral`.
- `periodo` (string): rótulo do período, ex: `T1-2025`.
- `periodo_inicio` (date ISO): `YYYY-MM-DD`.
- `periodo_fim` (date ISO): `YYYY-MM-DD`.
- `periodo_status` (string): `encerrado` | `em_andamento`.
- `duracao_dias` (int): dias inclusivos do período.
- `unidade_sigla` (string): sigla da unidade ou `N.I.`.
- `unidade_nome` (string): nome da unidade ou `N.I.`.
- `id_entrega` (int): identificador da entrega no `planos_entregas_entregas`.
- `nome_entrega` (string): rótulo da entrega calculado com `COALESCE/NULLIF/TRIM`.
- `id_plano_entrega` (int): id do `planos_entregas` dono do plano.
- `inicio_vigencia_plano_entrega` (date): data de início do plano.
- `fim_vigencia_plano_entrega` (date): data de fim do plano.
- `total_horas_planejadas_entrega` (decimal): soma das horas proporcionais ponderadas por `forca_trabalho`, arredondada a 2 casas.
- `num_servidores_alocados` (int): `COUNT(DISTINCT plano_trabalho_id)`.

Formato do CSV: primeira linha = cabeçalho com as colunas acima; delimitador `|`; encoding `utf-8-sig`.

## 7. Auditoria, verificações automáticas e critérios de qualidade

Ao final da execução, o script (ou auditoria manual) deve executar os seguintes checks:

1. Existência de dados
     - `total_linhas > 0` — caso contrário, revisar filtros de período e `deleted_at`.

2. Proporção de zeros em `total_horas_planejadas_entrega`
     - Calcular por período: `pct_zero = count_if(total_horas_planejadas_entrega = 0) / total_linhas`.
     - Aviso se `pct_zero > 0.1` (10%) — investigar ausência de `carga_horaria` ou `forca_trabalho=0`.

3. Coerência entre `num_servidores_alocados` e `total_horas_planejadas_entrega`
     - Casos com `num_servidores_alocados = 0` mas `total_horas_planejadas_entrega > 0` indicam problema de agregação — investigar joins em `vinculos_ativos`.

4. Verificação de sobreposição temporal de PEs
     - Amostra: listar PEs com `CAST(pe.data_inicio AS DATE) > CAST(pe.data_fim AS DATE)` (erro de dados).

5. Consistência unidade/plano
     - `total_horas_planejadas_entrega` por `unidade_sigla` deve somar para o total do plano (validação por amostra de entregas grandes).

6. Alertas de qualidade de dados (impressos no console)
     - `ALERTA: X% linhas com total_horas_planejadas_entrega = 0`.
     - `AVISO: periodo em_andamento — resultados preliminares`.

## 8. Diagnósticos e queries auxiliares (QD — quick diagnostics)

Estas queries devem ser incluídas em `IND_07.4_diagnostico_*.py` quando houver divergências ou para apoiar validação manual.

- QD-01 — Contagem por unidade

```sql
SELECT un.sigla, COUNT(*) AS n_entregas
FROM petrvs_icmbio_planos_entregas_entregas pee
JOIN petrvs_icmbio_planos_entregas pe ON pe.id = pee.plano_entrega_id
JOIN petrvs_icmbio_planos_trabalhos_entregas pte ON pte.plano_entrega_entrega_id = pee.id
JOIN petrvs_icmbio_planos_trabalhos pt ON pt.id = pte.plano_trabalho_id
WHERE CAST(pe.data_inicio AS DATE) <= CAST('{fim}' AS DATE)
    AND CAST(pe.data_fim   AS DATE) >= CAST('{ini}' AS DATE)
    AND pee.deleted_at IS NULL
GROUP BY un.sigla
ORDER BY n_entregas DESC
```

- QD-02 — Breakdown por `forca_trabalho`

```sql
SELECT pte.forca_trabalho, COUNT(*) as n
FROM petrvs_icmbio_planos_trabalhos_entregas pte
JOIN petrvs_icmbio_planos_entregas_entregas pee ON pee.id = pte.plano_entrega_entrega_id
JOIN petrvs_icmbio_planos_entregas pe ON pe.id = pee.plano_entrega_id
WHERE CAST(pe.data_inicio AS DATE) <= CAST('{fim}' AS DATE)
    AND CAST(pe.data_fim   AS DATE) >= CAST('{ini}' AS DATE)
    AND pte.deleted_at IS NULL
GROUP BY pte.forca_trabalho
ORDER BY forca_trabalho DESC
```

- QD-03 — Entregas com `carga_horaria` ausente

```sql
SELECT pt.id, pt.unidade_id, pt.carga_horaria
FROM petrvs_icmbio_planos_trabalhos pt
WHERE pt.carga_horaria IS NULL
    AND CAST(pt.data_inicio AS DATE) <= CAST('{fim}' AS DATE)
    AND CAST(pt.data_fim   AS DATE) >= CAST('{ini}' AS DATE)
```

Use estes CSVs/queries como evidência no relatório de validação (A5) se necessário.

## 9. Exemplos de anomalias e como investigar

- Alta % de zeros: executar QD-03 e QD-02; se muitos `forca_trabalho=0`, perguntar à CGOV se há vínculos ativos sem dedicação alocada.
- Somas por unidade discrepantes: comparar agregação por `unidade_sigla` entre o CSV final e uma execução simplificada no DBeaver.
- Divergência com cálculo por dias úteis: documentar diferença e, se necessário, propor variante com aproximação de dias úteis (exportar calendário externo e juntar fora do Denodo).
