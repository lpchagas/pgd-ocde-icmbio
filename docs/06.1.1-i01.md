# I01 — Proporcao de Servidores por Regime de Trabalho

## 1. Finalidade do indicador

O I01 mostra **como a forca de trabalho do ICMBio se distribui entre os regimes de trabalho** no periodo analisado. Ele responde a pergunta:

**"Como os servidores estao distribuidos entre presencial, parcial, integral e outros regimes no periodo?"**

Este indicador tem **duas visoes complementares**:

- **Visao institucional (geral):** uma linha por regime, com o total e o percentual de servidores do ICMBio em cada modalidade.
- **Visao por unidade:** uma linha por combinacao unidade x regime, com o percentual calculado dentro de cada unidade.

> Nota tecnica (confirmada em 23.05.2026): a view `tipos_modalidades` esta inacessivel no Denodo. Por isso o regime e obtido via `integracao_servidores.modalidade_pgd` (campo SIAPE/PGD), com o caminho `planos_trabalhos -> usuarios -> integracao_servidores`.

## 2. Consulta completa (script Python, sem credenciais)

Abaixo esta o script completo de referencia, com credenciais mascaradas. Preencha os campos `<HOST>`, `<PORT>`, `<DATABASE>`, `<USER>` e `<PASS>` apenas no ambiente local.

```python
"""IND_01.1_run_total.py
Serie temporal mensal do I01 — Proporcao de Servidores por Regime de Trabalho.

Estrategia: uma unica consulta ao Denodo carrega todos os planos ativos com suas
datas de vigencia e modalidade. A segmentacao por mes e feita em Python, sem
multiplas chamadas ao banco (eficiente para ~14 mil registros).

Granularidade: MENSAL
Justificativa:
  - Regime de trabalho e uma condicao estrutural: muda quando o servidor abre novo
    plano ou troca de modalidade, o que ocorre ao longo do ano.
  - Escala mensal captura transicoes intra-anuais (portarias, mudancas de politica)
    sem o ruido de granularidade semanal.
  - Alinhada a frequencia de relatorios mensais de gestao do ICMBio.
  - Alternativas descartadas: trimestral (perde resolucao de mudancas de regime),
    anual (nao gera serie util), semanal (ruidosa — planos tem vigencia de meses).

Saidas (salvas em OUT_DIR):
  IND_01.2_v1_proporcao_mensal_AAAAMMDD_HHMM.csv      — serie institucional
  IND_01.2_v2_proporcao_unidade_mensal_AAAAMMDD_HHMM.csv — serie por unidade

Nota tecnica:
  - A view tipos_modalidades esta inacessivel no Denodo. Usar integracao_servidores.modalidade_pgd.
  - Alertas de qualidade: UUIDs sem rotulo de modalidade, periodos parciais (ultimo mes).
"""
import calendar
import csv
import os
import re
import sys
from collections import defaultdict
from datetime import date, datetime

import jpype
import jpype.imports

sys.stdout.reconfigure(encoding="utf-8", errors="replace")

# ── Conexao Denodo ─────────────────────────────────────────────────────────────
JAR = (
    r"C:\Users\<SEU_USUARIO>\AppData\Roaming\DBeaverData"
    r"\drivers\remote\drivers\jdbc\9\denodo-vdp-jdbcdriver.jar"
)
JVM_DLL = r"C:\Program Files\DBeaver\jre\bin\server\jvm.dll"
JDBC_URL = "jdbc:denodo://<HOST>:<PORT>/<DATABASE>"
USER = "<USER>"
PASS = "<PASS>"

OUT_DIR = os.path.dirname(os.path.abspath(__file__))

# ── Consulta principal ─────────────────────────────────────────────────────────
SQL_PLANOS = """
SELECT
    pt.usuario_id,
    COALESCE(un.sigla, 'N.I.') AS unidade_sigla,
    COALESCE(un.nome,  'N.I.') AS unidade_nome,
    CASE WHEN NULLIF(TRIM(COALESCE(ins.modalidade_pgd, '')), '') IS NULL
         THEN 'N.I.'
         ELSE TRIM(ins.modalidade_pgd)
    END                        AS modalidade,
    CAST(pt.data_inicio AS DATE) AS plano_inicio,
    CAST(pt.data_fim    AS DATE) AS plano_fim
FROM petrvs_icmbio_planos_trabalhos pt
JOIN petrvs_icmbio_usuarios u
    ON  u.id = pt.usuario_id
JOIN (
    SELECT cpf,
           MIN(NULLIF(TRIM(COALESCE(modalidade_pgd, '')), '')) AS modalidade_pgd
    FROM petrvs_icmbio_integracao_servidores
    WHERE cpf IS NOT NULL
    GROUP BY cpf
) ins ON ins.cpf = u.cpf
LEFT JOIN petrvs_icmbio_unidades un
    ON  un.id = pt.unidade_id
WHERE pt.deleted_at  IS NULL
  AND pt.usuario_id  IS NOT NULL
  AND pt.data_inicio IS NOT NULL
  AND pt.data_fim    IS NOT NULL
"""


def start_jvm():
    if not jpype.isJVMStarted():
        jpype.startJVM(JVM_DLL, classpath=[JAR])
        print("JVM iniciada.")


def get_conn():
    DM = jpype.JClass("java.sql.DriverManager")
    P  = jpype.JClass("java.util.Properties")()
    P.setProperty("user", USER)
    P.setProperty("password", PASS)
    return DM.getConnection(JDBC_URL, P)


def clean(val):
    if val is None:
        return "N.I."
    return re.sub(r"[\r\n]+", " / ", str(val)).strip() or "N.I."


def parse_date(val):
    if val is None:
        return None
    s = str(val).strip()
    try:
        return date.fromisoformat(s[:10])
    except (ValueError, IndexError):
        return None


def month_periods(first, last):
    today = date.today()
    cur = date(first.year, first.month, 1)
    while cur <= last:
        end_day = calendar.monthrange(cur.year, cur.month)[1]
        p_end = min(date(cur.year, cur.month, end_day), today)
        yield (
            f"{cur.year}-{cur.month:02d}",
            cur,
            p_end,
        )
        next_month = cur.month % 12 + 1
        next_year  = cur.year + (1 if cur.month == 12 else 0)
        cur = date(next_year, next_month, 1)


def save_csv(path, cols, rows):
    with open(path, "w", newline="", encoding="utf-8-sig") as f:
        writer = csv.writer(f, delimiter="|")
        writer.writerow(cols)
        writer.writerows(rows)
    print(f"  Salvo: {os.path.basename(path)}")


def detect_uuid(value):
    """Retorna True se value parece um UUID bruto (sem rotulo textual)."""
    import re as _re
    return bool(_re.fullmatch(
        r"[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}",
        str(value).strip().lower(),
    ))


def main():
    start_jvm()
    conn = get_conn()
    print("Conexao Denodo OK.\n")

    print("Carregando planos de trabalho do Denodo (sem filtro de periodo)...")
    stmt = conn.createStatement()
    rs   = stmt.executeQuery(SQL_PLANOS)

    planos, skipped = [], 0
    while rs.next():
        uid   = str(rs.getObject(1)).strip() if rs.getObject(1) else ""
        usig  = clean(rs.getObject(2))
        unome = clean(rs.getObject(3))
        mod   = clean(rs.getObject(4))
        ini   = parse_date(rs.getObject(5))
        fim   = parse_date(rs.getObject(6))
        if not uid or not ini or not fim or ini > fim:
            skipped += 1
            continue
        planos.append((uid, usig, unome, mod, ini, fim))

    rs.close()
    stmt.close()
    conn.close()

    if not planos:
        print("Nenhum dado encontrado. Encerrando.")
        return

    # Horizonte padrao: 01/01/2025 ate a data atual
    min_dt  = date(2025, 1, 1)
    max_dt  = date.today()
    periods = list(month_periods(min_dt, max_dt))

    stamp = datetime.now().strftime("%Y%m%d_%H%M")

    # V1 — Serie mensal institucional
    v1_cols = [
        "periodo", "periodo_inicio", "periodo_fim",
        "modalidade", "total_servidores", "proporcao_perc",
    ]
    v1_rows = []
    for label, p_start, p_end in periods:
        counts = defaultdict(set)
        for uid, _usig, _unome, mod, ini, fim in planos:
            if ini <= p_end and fim >= p_start:
                counts[mod].add(uid)
        if not counts:
            continue
        total = sum(len(s) for s in counts.values())
        for mod in sorted(counts, key=lambda m: -len(counts[m])):
            n   = len(counts[mod])
            pct = round(n * 100.0 / total, 2) if total else 0.0
            v1_rows.append([label, str(p_start), str(p_end), mod, n, pct])

    f1 = os.path.join(OUT_DIR, f"IND_01.2_v1_proporcao_mensal_{stamp}.csv")
    save_csv(f1, v1_cols, v1_rows)

    # V2 — Serie mensal por unidade
    v2_cols = [
        "periodo", "periodo_inicio", "periodo_fim",
        "unidade_sigla", "unidade_nome",
        "modalidade", "total_servidores", "proporcao_na_unidade_perc",
    ]
    v2_rows = []
    for label, p_start, p_end in periods:
        unit_counts = defaultdict(lambda: defaultdict(set))
        unit_names  = {}
        for uid, usig, unome, mod, ini, fim in planos:
            if ini <= p_end and fim >= p_start:
                unit_counts[usig][mod].add(uid)
                unit_names[usig] = unome
        for usig in sorted(unit_counts):
            unome      = unit_names[usig]
            total_unit = sum(len(s) for s in unit_counts[usig].values())
            for mod in sorted(unit_counts[usig], key=lambda m: -len(unit_counts[usig][m])):
                n   = len(unit_counts[usig][mod])
                pct = round(n * 100.0 / total_unit, 2) if total_unit else 0.0
                v2_rows.append([
                    label, str(p_start), str(p_end),
                    usig, unome, mod, n, pct,
                ])

    f2 = os.path.join(OUT_DIR, f"IND_01.2_v2_proporcao_unidade_mensal_{stamp}.csv")
    save_csv(f2, v2_cols, v2_rows)


if __name__ == "__main__":
    main()
```

## 3. Passos da consulta (explicacao detalhada)

1. **Extrair a base de planos de trabalho.**
    O script inicia com uma unica consulta ao Denodo que carrega todos os planos com seus campos de datas e servidor. Faz `JOIN` com `usuarios` para obter o CPF e com `integracao_servidores` para recuperar o campo `modalidade_pgd`. Essa abordagem e mais eficiente que multiplos filtros por periodo — reduz latencia de rede em ~14 mil registros.

2. **Normalizar e validar dados.**
    Campos de unidade (`unidade_sigla`, `unidade_nome`) e modalidade passam por limpeza (`TRIM`, substituicao de nulos por `N.I.`). Descartar planos com datas invalidas (`data_inicio` nula, `data_fim` nula, ou `data_inicio > data_fim`). Essa filtragem evita distorcoes na contagem mensal.

3. **Gerar a serie mensal de periodos.**
    A funcao `month_periods(first, last)` gera uma lista de (label, p_start, p_end) para cada mes calendario do intervalo. Cada periodo e limitado pela data atual (`date.today()`) — o ultimo mes pode ser incompleto se ainda estiver em andamento. Exemplo: em 2026-06-14, o periodo junho e truncado para 2026-06-14.

4. **Calcular sobreposicao plano × mes.**
    Para cada plano e cada mes, verificar se ha sobreposicao: `plano_inicio <= mes_fim AND plano_fim >= mes_inicio`. Esse teste permite que o mesmo servidor conte em multiplos meses quando seu plano atravessa meses (ex: um servidor com plano de 2025-01-15 a 2025-03-10 aparece em jan, fev e mar).

5. **Construir a visao institucional (V1).**
    Para cada mes, contar o numero de servidores **unicos** por modalidade (usando `set()` para evitar duplicatas) e calcular percentuais. Resultado: uma linha por combinacao mes × modalidade, com total e proporcao.

6. **Construir a visao por unidade (V2).**
    Para cada mes e unidade, contar servidores unicos e calcular percentuais internos a unidade. Resultado: uma linha por combinacao mes × unidade × modalidade.

7. **Emitir alertas de qualidade.**
    - **Alerta 1 — UUIDs sem rotulo:** detecta modalidades que sao UUIDs brutos (ex: `b08c34bd-c751-431c-9357-f3bc669f16cd`). Esses valores indicam que a integracao nao conseguiu vincular o codigo de modalidade a um rotulo. A recomendacao e mapear os UUIDs com a equipe de cadastro do PGD.
    - **Alerta 2 — Periodo parcial:** avisa se o ultimo periodo da serie e incompleto (quando `periodo_fim < ultimo_dia_do_mes`). Dados de periodos parciais sao preliminares — nao devem ser comparados com meses encerrados.

8. **Exportar dois CSVs com delimitador seguro.**
    Os arquivos sao gerados em `|` (pipe) para evitar conflito com textos administrativos (descricoes de entrega frequentemente contem `;` e `\n`). Inclui datas de inicio/fim do mes para facilitar filtros no Excel e BI. Imprime resumo: periodo coberto, total de linhas, nomes de arquivos.

## 4. Consulta de diagnostico pre-execucao (QT-01/02/03)

Execute as tres consultas abaixo no Denodo antes de gerar o I01. Elas confirmam o padrao temporal e ajudam a interpretar os resultados.

### QT-01 — Distribuicao de planos de entrega por ciclo

```sql
SELECT
    EXTRACT(YEAR  FROM CAST(pe.data_inicio AS DATE)) AS ano,
    EXTRACT(MONTH FROM CAST(pe.data_inicio AS DATE)) AS mes_inicio,
    EXTRACT(MONTH FROM CAST(pe.data_fim   AS DATE))  AS mes_fim,
    COUNT(*)                                          AS total_planos,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct_total
FROM planos_entregas pe
WHERE pe.deleted_at IS NULL
  AND CAST(pe.data_inicio AS DATE) >= CAST('2025-01-01' AS DATE)
GROUP BY
    EXTRACT(YEAR  FROM CAST(pe.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pe.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pe.data_fim   AS DATE))
ORDER BY ano, mes_inicio
```

**Por que usar:** confirma o padrao de ciclos do Eixo 2 e ajuda a manter consistencia entre eixos.

### QT-02 — Distribuicao de planos de trabalho por ciclo (mais relevante para o I01)

```sql
SELECT
    EXTRACT(YEAR  FROM CAST(pt.data_inicio AS DATE)) AS ano,
    EXTRACT(MONTH FROM CAST(pt.data_inicio AS DATE)) AS mes_inicio,
    EXTRACT(MONTH FROM CAST(pt.data_fim   AS DATE))  AS mes_fim,
    COUNT(*)                                          AS total_planos,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct_total
FROM planos_trabalhos pt
WHERE pt.deleted_at IS NULL
  AND CAST(pt.data_inicio AS DATE) >= CAST('2025-01-01' AS DATE)
GROUP BY
    EXTRACT(YEAR  FROM CAST(pt.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pt.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pt.data_fim   AS DATE))
ORDER BY ano, mes_inicio
```

**Por que usar:** mostra como os planos de trabalho se distribuem ao longo do tempo e ajuda a confirmar se a segmentacao trimestral/quadrimestral faz sentido.

### QT-03 — Distribuicao de avaliacoes por mes de referencia

```sql
SELECT
    EXTRACT(YEAR  FROM CAST(a.data_avaliacao AS DATE)) AS ano,
    EXTRACT(MONTH FROM CAST(a.data_avaliacao AS DATE)) AS mes,
    COUNT(*)                                            AS total_avaliacoes
FROM avaliacoes a
WHERE a.deleted_at IS NULL
  AND CAST(a.data_avaliacao AS DATE) >= CAST('2025-01-01' AS DATE)
GROUP BY
    EXTRACT(YEAR  FROM CAST(a.data_avaliacao AS DATE)),
    EXTRACT(MONTH FROM CAST(a.data_avaliacao AS DATE))
ORDER BY ano, mes
```

**Por que usar:** valida a distribuicao temporal do Eixo 4, mantendo coerencia entre eixos no relatorio final.

## 6. Dicionario de dados (colunas do CSV)

O I01 gera dois CSVs. Abaixo o significado de cada coluna em formato de tabela.

**V1 — proporcao mensal institucional** (`IND_01.2_v1_proporcao_mensal_*.csv`)

| Coluna | Descricao |
| --- | --- |
| `periodo` | Mes de referencia no formato AAAA-MM. Use esta coluna para ordenar a serie temporal e construir graficos mensais. |
| `periodo_inicio` | Primeiro dia do mes. Serve como ancora para filtros de intervalo e para consolidar meses parciais. |
| `periodo_fim` | Ultimo dia do mes (limitado a data atual quando o mes esta em andamento). Ajuda a identificar mes incompleto. |
| `modalidade` | Regime de trabalho (valor vindo de `integracao_servidores.modalidade_pgd`). Quando estiver `N.I.`, indica falta de preenchimento no cadastro. |
| `total_servidores` | Quantidade de servidores com plano ativo no mes e nessa modalidade. A soma das modalidades no mes gera o total institucional. |
| `proporcao_perc` | Percentual da modalidade no total institucional do mes. E o indicador principal para leitura de tendencia de regime. |

**V2 — proporcao mensal por unidade** (`IND_01.2_v2_proporcao_unidade_mensal_*.csv`)

| Coluna | Descricao |
| --- | --- |
| `periodo` | Mes de referencia no formato AAAA-MM. Permite comparar a evolucao da unidade ao longo do tempo. |
| `periodo_inicio` | Primeiro dia do mes. Utilize para filtros temporais no Excel ou BI. |
| `periodo_fim` | Ultimo dia do mes (limitado a data atual quando o mes esta em andamento). Mes com fim parcial deve ser interpretado com cautela. |
| `unidade_sigla` | Sigla oficial da unidade. Use para filtros rapidos e comparacoes entre unidades. |
| `unidade_nome` | Nome completo da unidade. Facilita leitura por gestores nao familiarizados com siglas. |
| `modalidade` | Regime de trabalho da unidade no mes (ou `N.I.` quando nao informado). |
| `total_servidores` | Quantidade de servidores ativos na unidade e modalidade no mes. Serve de base para a proporcao. |
| `proporcao_na_unidade_perc` | Percentual da modalidade dentro da unidade no mes. Permite comparar o perfil da unidade com a media institucional. |

## 7. Como interpretar o resultado (exemplos reais)

Os exemplos abaixo usam dados reais do CSV `IND_01.2_v2_proporcao_unidade_mensal_20260524_1029.csv`. Para uso gerencial, leia o I01 como fotografia da **estrutura de regime**: ele indica o ambiente de trabalho predominante e ajuda a calibrar metas, acompanhamento e comparacoes entre unidades.

### Exemplo 1 — Distribuicao institucional (jan/2025)

No periodo 2025-01, a distribuicao institucional foi:

- **presencial:** 64,24%
- **parcial:** 25,91%
- **integral:** 6,08%
- **N.I.:** 1,39%

Isso indica que, naquele mes, a maior parte dos servidores estava em regime presencial. Para a gestao, esse dado reforca a necessidade de mecanismos de acompanhamento presencial e rotinas de campo, pois a maioria da forca de trabalho estava fisicamente nas unidades.

### Exemplo 2 — Unidade ACADEBIO (jan/2025)

A unidade **ACADEBIO** teve:

- presencial: 50,00%
- parcial: 35,71%
- integral: 7,14%

Isso mostra um perfil mais equilibrado que a media institucional. Na pratica, a unidade combina diferentes regimes e pode exigir modelos de comunicacao e acompanhamento mais hibridos.

### Exemplo 3 — Unidade APA-BALEFRA (jan/2025)

A unidade **APA-BALEFRA** teve:

- presencial: 75,00%
- parcial: 25,00%

Aqui o regime presencial domina claramente. Gestores podem interpretar isso como unidade de perfil operacional e de campo, onde a mobilidade e a presencialidade tendem a ser parte do trabalho cotidiano.

### Exemplo 4 — Unidade APACARSTELSANTA (jan/2025)

### Importante sobre modalidades desconhecidas

Se aparecerem valores como `b08c34bd-c751-431c-9357-f3bc669f16cd`, isso significa **codigo de modalidade sem rotulo** na integracao. Nesses casos, o ideal e mapear o codigo com a equipe responsavel pelo cadastro do PGD.
