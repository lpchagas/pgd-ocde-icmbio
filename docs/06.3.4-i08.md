# I08 — Proporção de Horas por Entrega — Planejadas (%)

## 1. Finalidade do indicador

O I08 calcula **qual percentual da capacidade total de trabalho de cada unidade foi planejado para cada entrega**, expressando em termos relativos o que o I07 expressa em termos absolutos.

A pergunta central que o I08 responde é: **"Qual é o peso relativo de cada entrega no esforço total planejado da unidade — em percentual da sua capacidade disponível?"**

O resultado traz uma linha por entrega com:

- **`horas_planejadas_entrega`** — horas alocadas àquela entrega (mesmo valor do I07)
- **`total_horas_disponiveis_unidade`** — soma de todas as horas de todos os planos de trabalho ativos da unidade no período
- **`proporcao_horas_perc`** — `horas_planejadas_entrega / total_horas_disponiveis_unidade × 100`

A diferença central em relação ao I07: o I07 compara entregas em valores absolutos dentro da mesma unidade; o I08 normaliza pelo total da unidade, permitindo **comparar o peso relativo de entregas entre unidades de capacidades diferentes**. Uma entrega que absorve 400 horas em uma unidade com capacidade de 800 horas (50%) é proporcionalmente mais pesada do que uma entrega de 800 horas em uma unidade com 4.000 horas disponíveis (20%).

## 2. Consulta completa (MySQL)

```sql
with recursive
parametros as (
    select
        date('2025-01-01') as data_inicio,
        date('2025-12-31') as data_fim,
        8                  as horas_por_dia,
        0                  as incluir_excluidos
),
anos as (
    select year((select data_inicio from parametros)) as ano
    union all
    select ano + 1
    from anos
    where ano < year((select data_fim from parametros))
),
feriados_fixos as (
    select date(concat(ano, '-01-01')) as data_feriado from anos
    union all select date(concat(ano, '-04-21')) from anos
    union all select date(concat(ano, '-05-01')) from anos
    union all select date(concat(ano, '-09-07')) from anos
    union all select date(concat(ano, '-10-12')) from anos
    union all select date(concat(ano, '-11-02')) from anos
    union all select date(concat(ano, '-11-15')) from anos
    union all select date(concat(ano, '-11-20')) from anos
    union all select date(concat(ano, '-12-25')) from anos
),
feriados_moveis as (
    select null as data_feriado where 1 = 0
    -- union all select date('2025-04-18')  -- Sexta-feira da Paixão 2025
    -- union all select date('2026-04-03')  -- Sexta-feira da Paixão 2026
),
feriados_nacionais as (
    select data_feriado from feriados_fixos
    union
    select data_feriado from feriados_moveis
),
calendario as (
    select (select data_inicio from parametros) as data_dia
    union all
    select date_add(data_dia, interval 1 day)
    from calendario
    where data_dia < (select data_fim from parametros)
),
links_distintos as (
    select
        pte.plano_trabalho_id,
        pte.plano_entrega_entrega_id    as id_entrega,
        coalesce(pte.forca_trabalho, 0) as forca_trabalho
    from planos_trabalhos_entregas pte
    where pte.plano_entrega_entrega_id is not null
),
horas_planejadas_por_plano as (
    select
        pt.id          as plano_trabalho_id,
        pt.unidade_id,
        count(c.data_dia) * p.horas_por_dia as horas_planejadas_plano
    from planos_trabalhos pt
    cross join parametros p
    join calendario c
        on c.data_dia between greatest(date(pt.data_inicio), p.data_inicio)
                          and least(date(pt.data_fim),   p.data_fim)
    left join feriados_nacionais fn
        on fn.data_feriado = c.data_dia
    where date(pt.data_inicio) <= p.data_fim
      and date(pt.data_fim)   >= p.data_inicio
      and (p.incluir_excluidos = 1 or pt.deleted_at is null)
      and dayofweek(c.data_dia) not in (1, 7)
      and fn.data_feriado is null
    group by pt.id, pt.unidade_id, p.horas_por_dia
),
horas_alocadas_por_entrega as (
    select
        coalesce(un.sigla, 'N.I.') as unidade_sigla,
        coalesce(un.nome, 'N.I.')  as unidade_nome,
        coalesce(
            nullif(trim(coalesce(pee.descricao, '')), ''),
            nullif(trim(coalesce(pee.descricao_entrega, '')), ''),
            'N.I.'
        )                          as nome_entrega,
        coalesce(
            nullif(trim(coalesce(pee.descricao_entrega, '')), ''),
            'N.I.'
        )                          as descricao_entrega,
        hpp.horas_planejadas_plano * (ld.forca_trabalho / 100.0) as horas_planejadas_alocadas
    from links_distintos ld
    join horas_planejadas_por_plano hpp
        on hpp.plano_trabalho_id = ld.plano_trabalho_id
    left join planos_entregas_entregas pee
        on pee.id = ld.id_entrega
    left join unidades un
        on un.id = hpp.unidade_id
),
capacidade_unidade as (
    select
        coalesce(un.sigla, 'N.I.')      as unidade_sigla,
        sum(hpp.horas_planejadas_plano) as total_horas_disponiveis_unidade
    from horas_planejadas_por_plano hpp
    left join unidades un
        on un.id = hpp.unidade_id
    group by coalesce(un.sigla, 'N.I.')
)
select
    h.unidade_sigla,
    min(h.unidade_nome)                        as unidade_nome,
    h.nome_entrega,
    min(h.descricao_entrega)                   as descricao_entrega,
    round(sum(h.horas_planejadas_alocadas), 2) as horas_planejadas_entrega,
    round(c.total_horas_disponiveis_unidade, 2) as total_horas_disponiveis_unidade,
    round(
        (sum(h.horas_planejadas_alocadas) / nullif(c.total_horas_disponiveis_unidade, 0)) * 100,
        2
    ) as proporcao_horas_perc
from horas_alocadas_por_entrega h
join capacidade_unidade c
    on c.unidade_sigla = h.unidade_sigla
group by h.unidade_sigla, h.nome_entrega, c.total_horas_disponiveis_unidade
order by h.unidade_sigla, proporcao_horas_perc desc;
```

> **Requisito técnico:** esta consulta usa `WITH RECURSIVE` e requer **MySQL 8.0 ou superior**. Para períodos superiores a ~2,7 anos, execute `SET SESSION cte_max_recursion_depth = 5000;` antes da query.

## 3. Passos da consulta

O I08 é uma extensão direta do I07. Os seis primeiros blocos (`parametros`, `anos`, `feriados_fixos`, `feriados_moveis`, `feriados_nacionais`, `calendario`, `links_distintos`, `horas_planejadas_por_plano`, `horas_alocadas_por_entrega`) são idênticos ou equivalentes aos do I07. O bloco que diferencia o I08 é o `capacidade_unidade`, e o `SELECT` final acrescenta a divisão pelo total da unidade.

### Passos 1 a 7 — Blocos compartilhados com o I07

Os blocos `parametros`, `anos`, `feriados_fixos`, `feriados_moveis`, `feriados_nacionais`, `calendario`, `links_distintos`, `horas_planejadas_por_plano` e `horas_alocadas_por_entrega` têm a mesma lógica documentada no [I07](06.3.3-i07.md). A única diferença estrutural em `horas_alocadas_por_entrega` é que o I08 não inclui as colunas de vigência do PE (`id_plano_entrega`, `inicio_vigencia_plano_entrega`, `fim_vigencia_plano_entrega`) — o agrupamento final é por nome de entrega e unidade, sem desdobrar por ciclo de PE.

Consulte o documento do I07 para a explicação detalhada desses blocos.

---

### Passo 8 — Bloco `capacidade_unidade`: denominador do percentual

```sql
capacidade_unidade as (
    select
        coalesce(un.sigla, 'N.I.')      as unidade_sigla,
        sum(hpp.horas_planejadas_plano) as total_horas_disponiveis_unidade
    from horas_planejadas_por_plano hpp
    left join unidades un on un.id = hpp.unidade_id
    group by coalesce(un.sigla, 'N.I.')
)
```

Este é o bloco exclusivo do I08. Ele soma o total de horas de **todos** os planos de trabalho da unidade no período — independentemente de qual entrega cada plano está vinculado. É o denominador da proporção.

**Por que partir de `horas_planejadas_por_plano` e não de `horas_alocadas_por_entrega`:** o total da unidade precisa refletir toda a capacidade disponível, incluindo:

- Planos de trabalho com `forca_trabalho = 0` (servidores sem alocação declarada)
- Planos de trabalho sem nenhuma entrega vinculada
- Horas de planos cujo vínculo com entregas foi removido

Se o denominador fosse a soma de `horas_alocadas_por_entrega`, ele excluiria essas situações e produziria um total de capacidade artificialmente menor — o que inflaria o percentual de cada entrega.

**Invariante esperada:** a soma de `proporcao_horas_perc` de todas as entregas de uma unidade deve ser menor ou igual a 100%. Ela será menor quando há horas de planos sem vínculo com entregas (o que é comum quando `forca_trabalho` está nulo ou zero em muitos vínculos). Se a soma ultrapassar 100%, indica que a soma dos `forca_trabalho` dos planos supera 100% — cenário de registro inválido identificado pela consulta de auditoria 2 do [Eixo 3](06.3-eixo3.md).

---

### Passo 9 — SELECT final: proporção de horas por entrega

```sql
select
    h.unidade_sigla,
    min(h.unidade_nome)                        as unidade_nome,
    h.nome_entrega,
    min(h.descricao_entrega)                   as descricao_entrega,
    round(sum(h.horas_planejadas_alocadas), 2) as horas_planejadas_entrega,
    round(c.total_horas_disponiveis_unidade, 2) as total_horas_disponiveis_unidade,
    round(
        (sum(h.horas_planejadas_alocadas) / nullif(c.total_horas_disponiveis_unidade, 0)) * 100,
        2
    ) as proporcao_horas_perc
from horas_alocadas_por_entrega h
join capacidade_unidade c on c.unidade_sigla = h.unidade_sigla
group by h.unidade_sigla, h.nome_entrega, c.total_horas_disponiveis_unidade
order by h.unidade_sigla, proporcao_horas_perc desc;
```

O `JOIN` interno com `capacidade_unidade` garante que apenas unidades com capacidade calculada apareçam no resultado. O `nullif(c.total_horas_disponiveis_unidade, 0)` previne divisão por zero para unidades com zero horas disponíveis — situação teórica, mas possível se todos os planos da unidade tiverem vigência fora do período.

O resultado é ordenado por unidade e por proporção decrescente — as entregas mais pesadas em termos relativos aparecem primeiro.

## 4. Como interpretar o resultado

### Estrutura do resultado

| Coluna | O que informa |
| --- | --- |
| `unidade_sigla` | Sigla da unidade |
| `unidade_nome` | Nome completo da unidade |
| `nome_entrega` | Nome da entrega (campo `descricao` com fallback para `descricao_entrega`) |
| `descricao_entrega` | Campo `descricao_entrega` como contexto adicional |
| `horas_planejadas_entrega` | Horas absolutas alocadas à entrega (mesmo valor do I07) |
| `total_horas_disponiveis_unidade` | Total de horas de todos os planos ativos da unidade no período |
| `proporcao_horas_perc` | Percentual da capacidade da unidade alocado a essa entrega |

### Exemplo com dados do ICMBio

Análise do período 2025-01-01 a 2025-12-31. Mesmo conjunto de dados do exemplo do I07:

| unidade_sigla | nome_entrega | horas_planejadas_entrega | total_horas_disponiveis_unidade | proporcao_horas_perc |
| --- | --- | --- | --- | --- |
| CGOV | Relatório anual de dimensionamento | 1.280,00 | 4.800,00 | 26,67 |
| CGOV | Mapeamento de processos críticos | 960,00 | 4.800,00 | 20,00 |
| CGOV | Nota técnica de avaliação institucional | 320,00 | 4.800,00 | 6,67 |
| COGEO | Atualização do banco geoespacial | 2.400,00 | 3.200,00 | 75,00 |
| COGEO | Modelagem de distribuição de espécies | 800,00 | 3.200,00 | 25,00 |
| AUDIT | Auditoria de contratos externos | 480,00 | 1.600,00 | 30,00 |
| AUDIT | Revisão de processos de licitação | 160,00 | 1.600,00 | 10,00 |

**Leituras caso a caso:**

**COGEO — "Atualização do banco geoespacial" (75%):** três quartos da capacidade disponível da COGEO está planejada para uma única entrega. Isso indica que a entrega é a prioridade absoluta da unidade no período — o que pode ser justificado (grandes projetos de banco de dados geoespacial têm essa natureza) ou pode indicar falta de diversificação do PE. O I06 revelará se essa concentração de horas reflete cobertura de equipe ou dependência de um único especialista.

**COGEO — soma das proporções = 100% (75% + 25%):** a capacidade total da unidade está inteiramente declarada nessas duas entregas, sem horas "sem alocação". Isso indica preenchimento exemplar dos campos `forca_trabalho` nos planos de trabalho da unidade.

**CGOV — soma das proporções = 53,34% (26,67% + 20,00% + 6,67%):** quase metade da capacidade disponível da CGOV (4.800 horas) não está alocada explicitamente em nenhuma entrega. Isso pode indicar: (a) entregas sem vínculos em `planos_trabalhos_entregas`; (b) vínculos com `forca_trabalho = 0`; (c) capacidade genuinamente disponível para atividades não estruturadas no PE. A consulta de auditoria 2 do Eixo 3 quantifica o preenchimento de `forca_trabalho` por unidade.

**AUDIT — "Auditoria de contratos externos" (30%) vs "Revisão de processos" (10%):** proporção de 3:1 em termos relativos — consistente com o I07. A soma é 40%, indicando que 60% da capacidade da AUDIT está em outras entregas não listadas ou sem alocação declarada.

**Comparação entre unidades:** em termos absolutos (I07), a COGEO tem a entrega com mais horas (2.400h vs 1.280h da CGOV). Em termos relativos (I08), a concentração na COGEO é mais extrema: a entrega principal consome 75% da capacidade da unidade, enquanto a entrega principal da CGOV consome 26,67%. Isso ilustra precisamente o valor do I08 sobre o I07: unidades menores (com menos horas totais disponíveis) tendem a ter proporções mais altas para suas principais entregas, mesmo quando o valor absoluto é menor.

### Nuances importantes

**`proporcao_horas_perc` pode ser interpretada como "prioridade de planejamento".** Uma entrega com 40% da capacidade da unidade é, em termos de planejamento formal, quatro vezes mais prioritária que uma entrega com 10%. Se essa proporção não refletir a importância estratégica real das entregas, o planejamento pode estar desalinhado das prioridades da unidade.

**A soma das proporções de uma unidade pode ser menor que 100% por razões legítimas.** Um servidor que declarou apenas 30% de `forca_trabalho` distribuído entre entregas dedicou os outros 70% a atividades não vinculadas a nenhuma entrega específica — reuniões, gestão, capacitação, suporte. Isso não é um erro; é uma parte esperada da jornada de trabalho que não se traduz em entregas formais.

**A soma pode superar 100% se `forca_trabalho` estiver com valores inconsistentes.** Se um servidor declarou `forca_trabalho = 80` para três entregas diferentes (soma = 240%), as horas serão multiplicadas por cada percentual individualmente, e o total de horas alocadas excederá as horas disponíveis. A consulta de auditoria 2 do Eixo 3 identifica vínculos com `forca_trabalho > 100`.

**Entregas sem nome aparecem como `'N.I.'`** e podem ser agrupadas em uma única linha se múltiplas entregas distintas tiverem nome vazio. O campo `id_entrega` foi omitido do I08 (para focar na visão por nome), mas se houver necessidade de distinguir essas entregas, adicione `ld.id_entrega` ao `SELECT` e ao `GROUP BY`.

| Situação observada no I08 | Próximo indicador | Pergunta a responder |
