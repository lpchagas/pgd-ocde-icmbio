# I12 — Coerência entre Avaliação do PT e do PE

## 1. Finalidade do indicador.

O I12 fecha o Eixo 4 comparando **duas perspectivas avaliativas sobre o mesmo objeto — a unidade organizacional — geradas por caminhos diferentes no PETRVS**.

A primeira perspectiva é a soma das avaliações individuais: os Planos de Trabalho de cada servidor da unidade são avaliados separadamente, e a média dessas notas representa a visão micro do desempenho. A segunda perspectiva é a avaliação coletiva: o Plano de Entregas da unidade recebe uma nota que representa a visão da equipe sobre o conjunto das suas próprias entregas.

Em uma organização com gestão de desempenho madura, as duas perspectivas devem ser **coerentes entre si** — se os servidores individualmente foram bem avaliados, a unidade como conjunto também deveria ter sido bem avaliada, e vice-versa. Quando há grande divergência entre as duas médias, isso é um sinal de que algo no processo avaliativo não está funcionando como deveria.

A pergunta central que o I12 responde é: **"A média das notas dos Planos de Trabalho individuais está alinhada com a nota do Plano de Entregas coletivo da mesma unidade?"**

O resultado é uma linha por unidade contendo as duas médias, a diferença absoluta entre elas, a direção da divergência e uma classificação de coerência. Esse conjunto permite:

- Detectar avaliação por cordialidade (PT com notas altas, PE com nota baixa);
- Identificar unidades onde o desempenho coletivo supera a soma das partes (PE melhor que os PTs individuais);
- Validar ou questionar as leituras dos indicadores I09, I10 e I11 — um I11 alto (muitos "Excepcional" em PT) com I12 divergente indica que o reconhecimento individual não é sustentado pela avaliação coletiva;
- Medir a **maturidade do processo avaliativo** da organização como um todo: quanto menor a divergência média entre PT e PE, mais coerente e confiável é o sistema de avaliação.

> **Pré-requisito:** este é o indicador do Eixo 4 mais sensível à configuração do PETRVS. Antes de executar, confirme com as Consultas 4 e 5 do [mapeamento do Eixo 4](06.4-eixo4.md) que os filtros `plano_trabalho_consolidacao_id is not null` e `plano_entrega_id is not null` identificam corretamente os dois tipos de avaliação no seu banco.

## 2. Consulta completa (MySQL).

```sql
with parametros as (
    select
        date('2025-01-01') as data_inicio,
        date('2025-12-31') as data_fim,
        0                  as incluir_excluidos
),
avaliacoes_pt as (
    select
        pt.unidade_id,
        tan.nota       as valor_nota
    from avaliacoes av
    join planos_trabalhos_consolidacoes ptc
        on ptc.id = av.plano_trabalho_consolidacao_id
    join planos_trabalhos pt
        on pt.id = ptc.plano_trabalho_id
    join tipos_avaliacoes_notas tan
        on tan.id = av.tipo_avaliacao_nota_id
    cross join parametros p
    where av.plano_trabalho_consolidacao_id is not null
      and (p.incluir_excluidos = 1 or av.deleted_at is null)
      and date(pt.data_inicio) <= p.data_fim
      and date(pt.data_fim)   >= p.data_inicio
      and (p.incluir_excluidos = 1 or pt.deleted_at is null)
),
avaliacoes_pe as (
    select
        pe.unidade_id,
        tan.nota       as valor_nota
    from avaliacoes av
    join planos_entregas pe
        on pe.id = av.plano_entrega_id
    join tipos_avaliacoes_notas tan
        on tan.id = av.tipo_avaliacao_nota_id
    cross join parametros p
    where av.plano_entrega_id is not null
      and (p.incluir_excluidos = 1 or av.deleted_at is null)
      and date(pe.data_inicio) <= p.data_fim
      and date(pe.data_fim)   >= p.data_inicio
      and (p.incluir_excluidos = 1 or pe.deleted_at is null)
),
media_pt_por_unidade as (
    select
        unidade_id,
        count(*)                       as total_avaliacoes_pt,
        round(avg(valor_nota), 2)      as media_nota_pt
    from avaliacoes_pt
    group by unidade_id
),
media_pe_por_unidade as (
    select
        unidade_id,
        count(*)                       as total_avaliacoes_pe,
        round(avg(valor_nota), 2)      as media_nota_pe
    from avaliacoes_pe
    group by unidade_id
),
coerencia as (
    select
        coalesce(un.sigla, 'N.I.')                                     as unidade_sigla,
        coalesce(un.nome, 'N.I.')                                      as unidade_nome,
        mpt.total_avaliacoes_pt,
        mpt.media_nota_pt,
        mpe.total_avaliacoes_pe,
        mpe.media_nota_pe,
        round(abs(mpt.media_nota_pt - mpe.media_nota_pe), 2)           as diferenca_absoluta,
        round(mpt.media_nota_pt - mpe.media_nota_pe, 2)                as diferenca_direcional
    from media_pt_por_unidade mpt
    join media_pe_por_unidade mpe
        on mpe.unidade_id = mpt.unidade_id
    left join unidades un
        on un.id = mpt.unidade_id
)
select
    unidade_sigla,
    unidade_nome,
    total_avaliacoes_pt,
    media_nota_pt,
    total_avaliacoes_pe,
    media_nota_pe,
    diferenca_absoluta,
    diferenca_direcional,
    case
        when diferenca_absoluta <= 1.0 then 'Coerente'
        when diferenca_absoluta <= 2.0 then 'Divergência moderada'
        else 'Alta divergência'
    end as classificacao_coerencia,
    case
        when diferenca_direcional > 0 then 'PT > PE'
        when diferenca_direcional < 0 then 'PE > PT'
        else 'Sem diferença'
    end as direcao_divergencia
from coerencia
order by diferenca_absoluta desc, unidade_sigla;
```

## 3. Passos da consulta.

O I12 tem a arquitetura mais extensa do Eixo 4: cinco CTEs de preparação mais o `SELECT` final. A estrutura é maior porque precisa construir e depois unir duas cadeias de dados que partem de tabelas-raiz diferentes.

### Passo 1 — Bloco `parametros`: painel de controle

```sql
parametros as (
    select
        date('2025-01-01') as data_inicio,
        date('2025-12-31') as data_fim,
        0                  as incluir_excluidos
)
```

Idêntico aos indicadores anteriores. O período de análise filtrará tanto os planos de trabalho (PT) quanto os planos de entregas (PE) — garantindo que as duas médias calculadas se refiram ao mesmo intervalo de tempo.

---

### Passo 2 — Bloco `avaliacoes_pt`: avaliações da perspectiva individual

```sql
avaliacoes_pt as (
    select
        pt.unidade_id,
        tan.nota       as valor_nota
    from avaliacoes av
    join planos_trabalhos_consolidacoes ptc
        on ptc.id = av.plano_trabalho_consolidacao_id
    join planos_trabalhos pt
        on pt.id = ptc.plano_trabalho_id
    join tipos_avaliacoes_notas tan
        on tan.id = av.tipo_avaliacao_nota_id
    cross join parametros p
    where av.plano_trabalho_consolidacao_id is not null
      and (p.incluir_excluidos = 1 or av.deleted_at is null)
      and date(pt.data_inicio) <= p.data_fim
      and date(pt.data_fim)   >= p.data_inicio
      and (p.incluir_excluidos = 1 or pt.deleted_at is null)
)
```

Idêntico ao bloco `avaliacoes_pt` do I09, I10 e I11. Percorre a cadeia `avaliacoes → planos_trabalhos_consolidacoes → planos_trabalhos → unidade_id`, retornando para cada avaliação de PT: a unidade à qual o servidor pertence e a nota numérica recebida.

---

### Passo 3 — Bloco `avaliacoes_pe`: avaliações da perspectiva coletiva

Este é o bloco novo e central do I12. Ele coleta as avaliações de Plano de Entregas — a perspectiva coletiva — usando uma cadeia de relacionamentos diferente da do PT.

```sql
avaliacoes_pe as (
    select
        pe.unidade_id,
        tan.nota       as valor_nota
    from avaliacoes av
    join planos_entregas pe
        on pe.id = av.plano_entrega_id
    join tipos_avaliacoes_notas tan
        on tan.id = av.tipo_avaliacao_nota_id
    cross join parametros p
    where av.plano_entrega_id is not null
      and (p.incluir_excluidos = 1 or av.deleted_at is null)
      and date(pe.data_inicio) <= p.data_fim
      and date(pe.data_fim)   >= p.data_inicio
      and (p.incluir_excluidos = 1 or pe.deleted_at is null)
)
```

**Por que a cadeia é mais curta para PE:**

No caso do PT, a avaliação está vinculada a uma consolidação, que por sua vez está vinculada ao plano de trabalho, que finalmente contém a `unidade_id`. São três saltos.

No caso do PE, a avaliação está vinculada diretamente ao plano de entregas (`plano_entrega_id`), que já contém a `unidade_id`. São apenas dois saltos:

```
Cadeia PT:  avaliacoes → planos_trabalhos_consolidacoes → planos_trabalhos → unidade_id
Cadeia PE:  avaliacoes → planos_entregas → unidade_id
```

**O filtro `plano_entrega_id is not null`:** o critério que identifica as avaliações de PE — o campo `plano_entrega_id` está preenchido. É o espelho exato do filtro `plano_trabalho_consolidacao_id is not null` do bloco anterior.

**O filtro de período usa as datas do plano de entregas** (`pe.data_inicio`, `pe.data_fim`), e não as datas da avaliação em si. Isso garante que o I12 compare avaliações que se referem ao mesmo ciclo de planejamento, e não simplesmente avaliações registradas no mesmo intervalo de datas.

---

### Passo 4 — Blocos `media_pt_por_unidade` e `media_pe_por_unidade`: calculando as duas médias

```sql
media_pt_por_unidade as (
    select
        unidade_id,
        count(*)                       as total_avaliacoes_pt,
        round(avg(valor_nota), 2)      as media_nota_pt
    from avaliacoes_pt
    group by unidade_id
),
media_pe_por_unidade as (
    select
        unidade_id,
        count(*)                       as total_avaliacoes_pe,
        round(avg(valor_nota), 2)      as media_nota_pe
    from avaliacoes_pe
    group by unidade_id
)
```

Dois blocos curtos e simétricos que condensam, cada um, o seu conjunto de avaliações em uma linha por unidade com o total e a média das notas. Separá-los em CTEs distintos — em vez de calcular as médias diretamente no bloco de junção — mantém a consulta legível e facilita diagnóstico em caso de resultados inesperados: você pode executar cada CTE isoladamente para verificar seus valores antes de comparar.

---

### Passo 5 — Bloco `coerencia`: unindo as duas médias e calculando a divergência

```sql
coerencia as (
    select
        coalesce(un.sigla, 'N.I.')                                     as unidade_sigla,
        coalesce(un.nome, 'N.I.')                                      as unidade_nome,
        mpt.total_avaliacoes_pt,
        mpt.media_nota_pt,
        mpe.total_avaliacoes_pe,
        mpe.media_nota_pe,
        round(abs(mpt.media_nota_pt - mpe.media_nota_pe), 2)           as diferenca_absoluta,
        round(mpt.media_nota_pt - mpe.media_nota_pe, 2)                as diferenca_direcional
    from media_pt_por_unidade mpt
    join media_pe_por_unidade mpe
        on mpe.unidade_id = mpt.unidade_id
    left join unidades un
        on un.id = mpt.unidade_id
)
```

Este bloco é onde as duas perspectivas se encontram. Para cada unidade, ele combina a média do PT com a média do PE e produz dois campos de diferença:

**`diferenca_absoluta` — o tamanho da divergência:**

```sql
round(abs(mpt.media_nota_pt - mpe.media_nota_pe), 2)
```

A função `abs()` retorna o valor absoluto (sem sinal), tornando a diferença sempre positiva. Uma diferença de 0.5 significa que as duas perspectivas se distanciam em meio ponto da escala — independentemente de qual é maior. Este é o valor usado para classificar a coerência.

**`diferenca_direcional` — o sentido da divergência:**

```sql
round(mpt.media_nota_pt - mpe.media_nota_pe, 2)
```

Sem o `abs()`, o valor pode ser positivo ou negativo — e o sinal revela **quem avalia melhor quem**:

- `diferenca_direcional > 0`: a média dos PTs individuais é maior que a nota do PE coletivo. Os avaliadores estão reconhecendo mais mérito nos servidores individualmente do que no conjunto da unidade.
- `diferenca_direcional < 0`: a nota do PE coletivo é maior que a média dos PTs. A unidade se avalia melhor como conjunto do que a soma das avaliações individuais sugere.
- `diferenca_direcional = 0`: perfeita coerência entre as duas perspectivas.

**O `join` entre PT e PE:** é um `join` interno (`join`, não `left join`). Isso significa que o I12 produz resultado **apenas para unidades que têm as duas perspectivas preenchidas** — tanto avaliações de PT quanto avaliação de PE no período. Unidades com apenas um dos dois tipos não aparecem no resultado, pois não é possível calcular a divergência sem os dois valores.

---

### Passo 6 — SELECT final: classificando a coerência e a direção

```sql
select
    unidade_sigla,
    unidade_nome,
    total_avaliacoes_pt,
    media_nota_pt,
    total_avaliacoes_pe,
    media_nota_pe,
    diferenca_absoluta,
    diferenca_direcional,
    case
        when diferenca_absoluta <= 1.0 then 'Coerente'
        when diferenca_absoluta <= 2.0 then 'Divergência moderada'
        else 'Alta divergência'
    end as classificacao_coerencia,
    case
        when diferenca_direcional > 0 then 'PT > PE'
        when diferenca_direcional < 0 then 'PE > PT'
        else 'Sem diferença'
    end as direcao_divergencia
from coerencia
order by diferenca_absoluta desc, unidade_sigla;
```

**As classificações de coerência:**

| Classificação | Condição | Interpretação |
| --- | --- | --- |
| Coerente | `diferenca_absoluta <= 1.0` | As duas perspectivas estão alinhadas — a visão individual e a coletiva convergem |
| Divergência moderada | `diferenca_absoluta <= 2.0` | Há diferença relevante entre as perspectivas — merece aprofundamento |
| Alta divergência | `diferenca_absoluta > 2.0` | As duas perspectivas contradizem uma à outra — sinal de disfunção avaliativa |

Os limiares de 1.0 e 2.0 foram definidos na especificação do *Performance Toolkit* da OCDE para a escala 1-5: em uma escala de 4 pontos de amplitude (de 1 a 5), uma diferença de 1 ponto representa 25% da escala — tolerável. Uma diferença acima de 2 pontos representa 50% da escala — contradição relevante.

**A ordenação `order by diferenca_absoluta desc`:** apresenta primeiro as unidades com maior divergência — os casos que demandam atenção analítica prioritária.

## 4. Como interpretar o resultado.

### Estrutura do resultado

| Coluna | O que informa |
| --- | --- |
| `unidade_sigla` / `unidade_nome` | Identidade da unidade |
| `total_avaliacoes_pt` | Quantas avaliações de PT sustentam a média individual |
| `media_nota_pt` | Média das notas dos Planos de Trabalho da unidade |
| `total_avaliacoes_pe` | Quantas avaliações de PE sustentam a média coletiva |
| `media_nota_pe` | Média das notas do Plano de Entregas da unidade |
| `diferenca_absoluta` | Tamanho da divergência — o indicador central |
| `diferenca_direcional` | Quem avalia melhor: sinal positivo = PT > PE; negativo = PE > PT |
| `classificacao_coerencia` | Coerente / Divergência moderada / Alta divergência |
| `direcao_divergencia` | `PT > PE`, `PE > PT` ou `Sem diferença` |

### Exemplo com dados do ICMBio

Resultado hipotético de 2025 com 6 unidades (as 5 dos exemplos anteriores + DINAP, usada para ilustrar Alta divergência):

| unidade_sigla | total_pt | media_pt | total_pe | media_pe | dif_abs | dif_dir | classificacao | direcao |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| DINAP | 30 | 4.80 | 3 | 2.50 | 2.30 | +2.30 | Alta divergência | PT > PE |
| DIREC | 12 | 2.42 | 2 | 3.50 | 1.08 | −1.08 | Divergência moderada | PE > PT |
| CGOV | 45 | 3.51 | 4 | 2.80 | 0.71 | +0.71 | Coerente | PT > PE |
| CGOF | 52 | 4.12 | 5 | 3.90 | 0.22 | +0.22 | Coerente | PT > PE |
| COGEO | 38 | 3.84 | 3 | 3.60 | 0.24 | +0.24 | Coerente | PT > PE |
| AUDIT | 18 | 3.00 | 2 | 3.20 | 0.20 | −0.20 | Coerente | PE > PT |

**Leitura caso a caso:**

**DPAE — Alta divergência (PT > PE, +2.30):** este é o caso mais grave do conjunto. A média de PT é 4.80 — quase todos os servidores foram avaliados como "Excepcional" individualmente. Mas a avaliação do Plano de Entregas da unidade é 2.50 — "Inadequado" no nível coletivo. A contradição é severa: como pode uma equipe de servidores quase todos excepcionais produzir um conjunto de entregas avaliado como inadequado? Esse padrão é o sinal mais claro de **avaliação por cordialidade no nível individual**: os gestores não quiseram prejudicar os servidores com notas baixas nos PTs, mas ao avaliar o PE coletivamente foram forçados a reconhecer que os resultados da unidade ficaram aquém. Requer investigação aprofundada e provavelmente revisão dos critérios de avaliação dos PTs.

**DIMAN — Divergência moderada (PE > PT, −1.08):** inversão rara e analiticamente interessante. Os PTs individuais têm média baixa (2.42), mas o PE coletivo foi avaliado com 3.50. Há duas leituras possíveis: (a) os servidores individualmente tiveram dificuldades, mas a unidade como conjunto conseguiu entregar — indicando que o trabalho coletivo compensou as limitações individuais; (b) quem avaliou o PE aplicou critérios mais generosos do que quem avaliou os PTs. Cruzar com I02 (taxa de cumprimento das entregas) ajuda a distinguir: se o cumprimento objetivo de metas foi alto, a leitura (a) é mais plausível.

**CGOV — Coerente (PT > PE, +0.71):** diferença dentro do limiar de coerência. É normal que a média de PT seja ligeiramente superior à do PE — a avaliação individual tende a ser mais generosa que a coletiva. Uma diferença de 0.71 ponto em uma escala de 5 é aceitável e esperada.

**COGED — Coerente, altamente alinhados:** diferenças de 0.22 e 0.24 indicam excelente coerência. Os servidores são avaliados individualmente em um patamar muito próximo ao que a unidade reconhece coletivamente. Este é o padrão de maturidade avaliativa esperado.

**AUDIT — Coerente (PE > PT, −0.20):** pequena inversão dentro do limiar de coerência. O PE (3.20) ficou marginalmente acima do PT (3.00), o que pode refletir que a auto-percepção coletiva da unidade é ligeiramente mais positiva que as avaliações individuais. Com todos os PTs em nota 3 (uso pro forma), a diferença de 0.20 é irrelevante — confirma o diagnóstico de ausência de diferenciação avaliativa na unidade.

### O I12 como validador dos demais indicadores do Eixo 4

O I12 é o indicador de fechamento do eixo porque permite contextualizar — e às vezes questionar — as leituras dos outros três:

| Situação | I09 | I11 | I12 | Diagnóstico |
| --- | --- | --- | --- | --- |
| Excelência genuína | Alta | Alto | Coerente (PT ≈ PE) | Confirma alto desempenho |
| Avaliação por cordialidade | Alta | Alto | Alta divergência (PT >> PE) | PT inflado; PE revela resultado real |
| Desempenho consistente médio | Médio | Baixo | Coerente | Uso honesto da escala, padrão adequado |
| Uso pro forma | ~3.0 | Zero | Coerente (dif. ≈ 0) | Avaliação sem diferenciação — processo cumprido formalmente |
| Problemas reconhecidos | Baixa | Zero | Coerente ou PE > PT | Dificuldades identificadas em ambas perspectivas |

### Unidades que não aparecem no resultado

O I12 usa `join` interno entre PT e PE. Unidades que aparecem nos indicadores I09-I11 mas não no I12 são unidades com avaliações de PT mas **sem avaliação de PE registrada** no período — ou vice-versa. Essas ausências são em si um dado relevante: indicam que o ciclo avaliativo não foi completado na sua dimensão coletiva (ou individual), o que é uma informação de gestão independente do conteúdo das avaliações.

Para identificar essas unidades, execute:

```sql
-- Unidades com PT avaliado mas sem PE avaliado no período
select distinct mpt.unidade_id
from media_pt_por_unidade mpt
left join media_pe_por_unidade mpe on mpe.unidade_id = mpt.unidade_id
where mpe.unidade_id is null;
```

> Esta consulta auxiliar deve ser executada após os CTEs `media_pt_por_unidade` e `media_pe_por_unidade`, dentro da mesma sessão SQL no DBeaver.
