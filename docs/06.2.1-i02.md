# I02 — Taxa de Cumprimento das Entregas por Unidade

## 1. Finalidade do indicador

O I02 mede **a proporcao de entregas concluidas em relacao ao total de entregas planejadas no ciclo** de cada unidade.

A pergunta central e: **"De todas as entregas registradas no Plano de Entregas (PE) da unidade, qual percentual foi efetivamente cumprido?"**

O criterio e binario:
- **Concluida:** `progresso_realizado >= progresso_esperado`
- **Nao concluida:** abaixo disso

Para transparencia, o resultado inclui duas visoes:

- **Criterio OCDE (principal):** usa todas as entregas do ciclo, inclusive de PEs ainda em avaliacao.
- **Criterio formal do PETRVS:** considera apenas entregas em PEs com status `AVALIADO` ou `CONCLUIDO`.

## 2. Consulta completa (script Python, sem credenciais)

Script de referencia com credenciais mascaradas. Preencha `<HOST>`, `<PORT>`, `<DATABASE>`, `<USER>` e `<PASS>` apenas no ambiente local.

```python
"""IND_02.1_run_total.py
Serie temporal do I02 — Taxa de Cumprimento das Entregas por Unidade.
"""
import csv
import os
import re
import sys
from datetime import date, datetime

import jpype
import jpype.imports

sys.stdout.reconfigure(encoding="utf-8", errors="replace")

# ── Conexao ─────────────────────────────────────────────────────────────────---
JAR = (
    r"C:\Users\leand\AppData\Roaming\DBeaverData"
    r"\drivers\remote\drivers\jdbc\9\denodo-vdp-jdbcdriver.jar"
)
JVM_DLL = r"C:\Program Files\DBeaver\jre\bin\server\jvm.dll"
JDBC_URL = "jdbc:denodo://<HOST>:<PORT>/<DATABASE>"
USER = "<USER>"
PASS = "<PASS>"

OUT_DIR = os.path.dirname(os.path.abspath(__file__))

SQL_I02 = """
WITH parametros AS (
    SELECT
        CAST('{ini}' AS DATE) AS data_inicio,
        CAST('{fim}' AS DATE) AS data_fim,
        0                     AS incluir_excluidos
),
universo_bruto AS (
    SELECT
        u.sigla     AS unidade_sigla,
        MIN(u.nome) AS unidade_nome,
        COUNT(*)    AS total_cadastradas
    FROM petrvs_icmbio_planos_entregas pe
    JOIN petrvs_icmbio_planos_entregas_entregas pee
        ON pee.plano_entrega_id = pe.id
    JOIN petrvs_icmbio_unidades u
        ON u.id = pe.unidade_id
    CROSS JOIN parametros p
    WHERE CAST(pe.data_inicio AS DATE) <= p.data_fim
      AND CAST(pe.data_fim   AS DATE) >= p.data_inicio
      AND (p.incluir_excluidos = 1 OR pe.deleted_at  IS NULL)
      AND (p.incluir_excluidos = 1 OR pee.deleted_at IS NULL)
    GROUP BY u.sigla
),
entregas_ciclo AS (
    SELECT
        u.sigla                              AS unidade_sigla,
        u.nome                               AS unidade_nome,
        pee.id                               AS id_entrega,
        pee.progresso_esperado               AS meta_planejada,
        COALESCE(pee.progresso_realizado, 0) AS meta_executada,
        CASE
            WHEN CAST(pee.data_fim AS DATE) BETWEEN p.data_inicio AND p.data_fim
            THEN 1 ELSE 0
        END                                  AS vence_no_periodo,
        CASE
            WHEN pe.status IN ('AVALIADO', 'CONCLUIDO') THEN 1 ELSE 0
        END                                  AS plano_avaliado
    FROM petrvs_icmbio_planos_entregas pe
    JOIN petrvs_icmbio_planos_entregas_entregas pee
        ON pee.plano_entrega_id = pe.id
    JOIN petrvs_icmbio_unidades u
        ON u.id = pe.unidade_id
    CROSS JOIN parametros p
    WHERE CAST(pe.data_inicio AS DATE) <= p.data_fim
      AND CAST(pe.data_fim   AS DATE) >= p.data_inicio
      AND (p.incluir_excluidos = 1 OR pe.deleted_at  IS NULL)
      AND (p.incluir_excluidos = 1 OR pee.deleted_at IS NULL)
      AND pee.progresso_esperado IS NOT NULL
      AND pee.progresso_esperado > 0
),
resumo AS (
    SELECT
        unidade_sigla,
        MIN(unidade_nome)                                                     AS unidade_nome,
        COUNT(*)                                                              AS total_no_ciclo,
        SUM(vence_no_periodo)                                                 AS total_vence_no_periodo,
        SUM(CASE WHEN meta_executada >= meta_planejada THEN 1 ELSE 0 END)     AS total_concluidas,
        ROUND(
            SUM(CASE WHEN meta_executada >= meta_planejada THEN 1 ELSE 0 END)
                * 100.0 / NULLIF(COUNT(*), 0),
            2
        )                                                                     AS taxa_cumprimento_perc,
        SUM(plano_avaliado)                                                   AS total_em_plano_avaliado,
        SUM(CASE WHEN plano_avaliado = 1 AND meta_executada >= meta_planejada
                 THEN 1 ELSE 0 END)                                           AS concluidas_em_plano_avaliado
    FROM entregas_ciclo
    GROUP BY unidade_sigla
)
SELECT
    r.unidade_sigla,
    r.unidade_nome,
    b.total_cadastradas,
    r.total_no_ciclo,
    r.total_vence_no_periodo,
    ROUND(r.total_vence_no_periodo * 100.0 / NULLIF(r.total_no_ciclo, 0), 1)
        AS proporcao_vence_no_periodo_perc,
    r.total_concluidas,
    r.taxa_cumprimento_perc,
    r.total_em_plano_avaliado,
    r.concluidas_em_plano_avaliado,
    CASE
        WHEN r.taxa_cumprimento_perc >= 90 THEN 'A — Alto desempenho'
        WHEN r.taxa_cumprimento_perc >= 70 THEN 'B — Bom desempenho'
        WHEN r.taxa_cumprimento_perc >= 50 THEN 'C — Desempenho intermediario'
        ELSE                                    'D — Baixo desempenho'
    END AS grupo_performance,
    CASE
        WHEN r.total_no_ciclo > r.total_em_plano_avaliado
        THEN 'atencao: ha entregas em planos nao avaliados'
        ELSE 'ciclo avaliado'
    END AS alerta_avaliacao
FROM resumo r
LEFT JOIN universo_bruto b ON b.unidade_sigla = r.unidade_sigla
ORDER BY r.taxa_cumprimento_perc DESC, r.unidade_sigla
"""


def build_periods():
    """2025 = trimestral | 2026+ = quadrimestral."
    today = date.today()
    raw = [
        ("T1-2025", "trimestral",    date(2025, 1,  1), date(2025, 3, 31)),
        ("T2-2025", "trimestral",    date(2025, 4,  1), date(2025, 6, 30)),
        ("T3-2025", "trimestral",    date(2025, 7,  1), date(2025, 9, 30)),
        ("T4-2025", "trimestral",    date(2025, 10, 1), date(2025, 12, 31)),
        ("Q1-2026", "quadrimestral", date(2026, 1,  1), date(2026, 4, 30)),
        ("Q2-2026", "quadrimestral", date(2026, 5,  1), date(2026, 8, 31)),
        ("Q3-2026", "quadrimestral", date(2026, 9,  1), date(2026, 12, 31)),
    ]
    periods = []
    for lbl, tipo, p_ini, p_fim in raw:
        if p_ini > today:
            continue
        status = "em_andamento" if p_fim >= today else "encerrado"
        periods.append((lbl, tipo, p_ini, p_fim, status))
    return periods


def start_jvm():
    if not jpype.isJVMStarted():
        jpype.startJVM(JVM_DLL, classpath=[JAR])
        print("JVM iniciada.")


def get_conn():
    DM = jpype.JClass("java.sql.DriverManager")
    P  = jpype.JClass("java.util.Properties")()
    P.setProperty("user", USER)
    P.setProperty("password", PASS)
    return DM.getConnection(JDBC_URL, P)


def clean(val):
    if val is None:
        return ""
    return re.sub(r"[\r\n]+", " / ", str(val)).strip()


def run_period(conn, ini, fim):
    sql = SQL_I02.format(ini=ini, fim=fim)
    stmt = conn.createStatement()
    rs   = stmt.executeQuery(sql)
    meta = rs.getMetaData()
    n    = meta.getColumnCount()
    cols = [str(meta.getColumnLabel(i + 1)) for i in range(n)]
    rows = []
    while rs.next():
        rows.append([clean(rs.getObject(i + 1)) for i in range(n)])
    rs.close()
    stmt.close()
    return cols, rows


def save_csv(path, cols, rows):
    with open(path, "w", newline="", encoding="utf-8-sig") as f:
        csv.writer(f, delimiter="|").writerows([cols] + rows)
    print(f"  Salvo: {os.path.basename(path)}")


def main():
    start_jvm()
    conn = get_conn()
    print("Conexao Denodo OK.\n")

    periods = build_periods()
    meta_cols = [
        "ciclo_tipo", "periodo", "periodo_inicio", "periodo_fim",
        "periodo_status", "duracao_dias",
    ]

    all_cols = None
    all_rows = []

    for lbl, tipo, p_ini, p_fim, status in periods:
        cols, rows = run_period(conn, p_ini, p_fim)
        if all_cols is None:
            all_cols = meta_cols + cols
        dur = (p_fim - p_ini).days + 1
        for row in rows:
            all_rows.append([tipo, lbl, str(p_ini), str(p_fim), status, dur] + row)

    conn.close()

    stamp = datetime.now().strftime("%Y%m%d_%H%M")
    fname = os.path.join(OUT_DIR, f"IND_02.2_taxa_cumprimento_temporal_{stamp}.csv")
    save_csv(fname, all_cols, all_rows)


if __name__ == "__main__":
    main()
```

## 3. Passos da consulta (explicacao detalhada)

1. **Definir os periodos de analise.**
    O script cria periodos a partir da regra institucional: 2025 trimestral e 2026 em diante quadrimestral. Cada periodo recebe um rotulo (ex: T1-2025) e um status (`encerrado` ou `em_andamento`), o que sera usado na leitura gerencial.

2. **Construir o universo bruto (`universo_bruto`).**
    Essa CTE conta todas as entregas registradas no ciclo, sem filtrar meta valida. O objetivo e medir o tamanho total do plano (linha de base), permitindo comparar quantas entregas ficam de fora por falta de meta.

3. **Selecionar as entregas elegiveis do ciclo (`entregas_ciclo`).**
    Aqui entram apenas entregas com `progresso_esperado > 0`. A CTE tambem marca se a entrega vence no periodo (`vence_no_periodo`) e se o PE ja foi avaliado (`plano_avaliado`).

4. **Agregar por unidade (`resumo`).**
    O calculo principal ocorre nesta etapa: total de entregas no ciclo, total concluidas e taxa de cumprimento. E aqui que se calcula a taxa OCDE (todas as entregas do ciclo) e os totais restritos a planos avaliados.

5. **Montar o resultado final com alertas.**
    A query final adiciona o grupo de performance (A/B/C/D) e o `alerta_avaliacao`, que sinaliza se ainda ha entregas em planos nao avaliados. Isso ajuda o gestor a separar resultados ja consolidados de resultados provisórios.

6. **Adicionar metadados de periodo no CSV.**
    O script prefixa cada linha com `ciclo_tipo`, `periodo`, `periodo_inicio`, `periodo_fim`, `periodo_status` e `duracao_dias`, permitindo comparar unidades ao longo do tempo e filtrar periodos em andamento.

7. **Gerar o CSV consolidado.**
    O arquivo final traz uma linha por unidade e por periodo. A analise gerencial deve sempre considerar o `periodo_status` para evitar conclusoes precipitadas em ciclos ainda abertos.

## 4. Consulta de diagnostico pre-execucao (QT-01/02/03)

Execute as tres consultas abaixo no Denodo antes de gerar o I02.

### QT-01 — Distribuicao de planos de entrega por ciclo

```sql
SELECT
    EXTRACT(YEAR  FROM CAST(pe.data_inicio AS DATE)) AS ano,
    EXTRACT(MONTH FROM CAST(pe.data_inicio AS DATE)) AS mes_inicio,
    EXTRACT(MONTH FROM CAST(pe.data_fim   AS DATE))  AS mes_fim,
    COUNT(*)                                          AS total_planos,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct_total
FROM planos_entregas pe
WHERE pe.deleted_at IS NULL
  AND CAST(pe.data_inicio AS DATE) >= CAST('2025-01-01' AS DATE)
GROUP BY
    EXTRACT(YEAR  FROM CAST(pe.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pe.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pe.data_fim   AS DATE))
ORDER BY ano, mes_inicio
```

**Por que usar:** confirma que 2025 e predominantemente trimestral e 2026 quadrimestral.

### QT-02 — Distribuicao de planos de trabalho por ciclo

```sql
SELECT
    EXTRACT(YEAR  FROM CAST(pt.data_inicio AS DATE)) AS ano,
    EXTRACT(MONTH FROM CAST(pt.data_inicio AS DATE)) AS mes_inicio,
    EXTRACT(MONTH FROM CAST(pt.data_fim   AS DATE))  AS mes_fim,
    COUNT(*)                                          AS total_planos,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct_total
FROM planos_trabalhos pt
WHERE pt.deleted_at IS NULL
  AND CAST(pt.data_inicio AS DATE) >= CAST('2025-01-01' AS DATE)
GROUP BY
    EXTRACT(YEAR  FROM CAST(pt.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pt.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pt.data_fim   AS DATE))
ORDER BY ano, mes_inicio
```

**Por que usar:** ajuda a entender se a vigencia dos planos de trabalho acompanha o ciclo do PE.

### QT-03 — Distribuicao de avaliacoes por mes de referencia

```sql
SELECT
    EXTRACT(YEAR  FROM CAST(a.data_avaliacao AS DATE)) AS ano,
    EXTRACT(MONTH FROM CAST(a.data_avaliacao AS DATE)) AS mes,
    COUNT(*)                                            AS total_avaliacoes
FROM avaliacoes a
WHERE a.deleted_at IS NULL
  AND CAST(a.data_avaliacao AS DATE) >= CAST('2025-01-01' AS DATE)
GROUP BY
    EXTRACT(YEAR  FROM CAST(a.data_avaliacao AS DATE)),
    EXTRACT(MONTH FROM CAST(a.data_avaliacao AS DATE))
ORDER BY ano, mes
```

**Por que usar:** confirma a distribuicao temporal do eixo de avaliacao para leitura conjunta.

## 6. Dicionario de dados (colunas do CSV)

O CSV do I02 inclui metadados de periodo e os campos do indicador. Colunas:

| Coluna | Descricao |
| --- | --- |
| `ciclo_tipo` | Tipo de ciclo do periodo (trimestral ou quadrimestral). Use para agrupar comparacoes em ciclos equivalentes. |
| `periodo` | Rotulo do periodo (ex: T1-2025). Facilita leitura e filtros no Excel. |
| `periodo_inicio` | Data inicial do periodo. Serve para recortes temporais e para ordenar o historico. |
| `periodo_fim` | Data final do periodo. Quando o periodo esta aberto, essa data ainda pode estar em andamento. |
| `periodo_status` | Indica se o periodo esta `encerrado` ou `em_andamento`. Resultados em andamento devem ser lidos com cautela. |
| `duracao_dias` | Duracao do periodo em dias. Ajuda a explicar comparacoes entre periodos com tamanhos diferentes. |
| `unidade_sigla` | Sigla da unidade. Use para filtros rapidos e dashboards. |
| `unidade_nome` | Nome completo da unidade. Facilita leitura gerencial e comunicacao institucional. |
| `total_cadastradas` | Entregas registradas no ciclo, sem filtro de meta. Indica o tamanho do plano da unidade. |
| `total_no_ciclo` | Entregas com meta valida (base do calculo). Diferenca entre `total_cadastradas` e `total_no_ciclo` indica metas mal preenchidas. |
| `total_vence_no_periodo` | Entregas cujo prazo individual vence dentro do periodo. Mostra se o ciclo tem entregas com vencimento no recorte. |
| `proporcao_vence_no_periodo_perc` | Percentual de entregas do ciclo que vencem no periodo. Ajuda a entender se o recorte pega apenas parte do ciclo. |
| `total_concluidas` | Entregas com `progresso_realizado >= progresso_esperado`. Base do criterio de conclusao OCDE. |
| `taxa_cumprimento_perc` | Indicador principal: concluidas / total_no_ciclo. E a taxa usada para comparacoes entre unidades. |
| `total_em_plano_avaliado` | Entregas em PEs com status `AVALIADO` ou `CONCLUIDO`. Representa o universo ja formalmente avaliado no PETRVS. |
| `concluidas_em_plano_avaliado` | Concluidas restritas a planos avaliados. Serve para leitura no criterio formal do PETRVS. |
| `grupo_performance` | Classificacao A/B/C/D a partir da taxa de cumprimento. Ajuda a sintetizar a leitura gerencial. |
| `alerta_avaliacao` | Alerta sobre planos nao avaliados. Indica que parte do resultado ainda pode mudar. |

## 7. Como interpretar o resultado (exemplos reais)

Exemplos abaixo usam o CSV `IND_02.2_taxa_cumprimento_temporal_20260524_1050.csv`. Para gestores, o I02 deve ser lido como **taxa de cumprimento do contrato da unidade**, sempre observando se o ciclo esta avaliado (`alerta_avaliacao`) e se a base e grande o suficiente para conclusoes seguras.

### Exemplo 1 — Unidade com 100% de cumprimento

No periodo **T1-2025**, a unidade **ACADEBIO** apresentou:

- `total_no_ciclo = 12`
- `total_concluidas = 12`
- `taxa_cumprimento_perc = 100.0`

Isso indica que todas as entregas com meta valida foram concluidas no ciclo. Para a gestao, esse e um resultado consistente, mas deve ser lido junto com `total_cadastradas` para saber se o volume de entregas e representativo.

### Exemplo 2 — Entregas que nao vencem no periodo

No mesmo periodo, a unidade **APAPIACABUCU** tem:

- `total_no_ciclo = 3`
- `total_vence_no_periodo = 0`
- `taxa_cumprimento_perc = 100.0`

Interpretacao: as entregas pertencem ao ciclo, mas nenhuma vence dentro do recorte. Mesmo assim, a taxa considera o ciclo inteiro (regra OCDE). Para o gestor, isso significa que a unidade pode ter prazos concentrados fora do periodo e, portanto, a taxa nao deve ser comparada diretamente com unidades que tem entregas vencendo no periodo.

### Exemplo 3 — Alerta de planos nao avaliados

A unidade **CECAV-SEDE** aparece com:

- `total_em_plano_avaliado = 0`
- `alerta_avaliacao = "atencao: ha entregas em planos nao avaliados"`

Interpretacao: o resultado e valido pelo criterio OCDE, mas ainda nao foi formalmente avaliado no PETRVS. Em termos gerenciais, a recomendacao e nao usar este numero para cobranca formal ate que os planos sejam avaliados.

### Exemplos adicionais

### Exemplo 4 — Unidade com ciclo totalmente avaliado

A unidade **CGADM** aparece com:

- `total_no_ciclo = 4`
- `taxa_cumprimento_perc = 100.0`
- `alerta_avaliacao = "ciclo avaliado"`

Interpretacao: todas as entregas do ciclo estao em planos avaliados, e o resultado e estavel. Este e o melhor cenario para comparacao entre unidades.

### Exemplo 5 — Unidade com alta taxa e base pequena

A unidade **CGFIN** apresenta:

- `total_no_ciclo = 2`
- `taxa_cumprimento_perc = 100.0`

Interpretacao: a taxa e alta, mas baseada em poucas entregas. Para a gestao, isso reduz a confiabilidade estatistica: uma unica entrega nao cumprida mudaria drasticamente o percentual.

### Exemplo 6 — Unidade com alerta e taxa alta

A unidade **CECAV-SEDE** tem:

- `taxa_cumprimento_perc = 100.0`
- `alerta_avaliacao = "atencao: ha entregas em planos nao avaliados"`

Interpretacao: o desempenho e alto, mas ainda sem validacao formal completa no PETRVS. O gestor deve acompanhar a avaliacao formal antes de consolidar a leitura em relatorios oficiais.
