# I03 — Taxa de Cumprimento de Metas por Entrega

## 1. Finalidade do indicador

O I03 detalha o que o I02 resume. Em vez de uma taxa unica por unidade, ele mostra **o percentual de execucao de cada entrega individual**.

Pergunta central:

**"Para cada entrega, qual percentual da meta foi efetivamente executado?"**

Este indicador traz **duas abordagens na mesma saida**:

- **Abordagem principal (ciclo):** calcula a taxa com base na parcela do ciclo (`progresso_esperado`).
- **Abordagem alternativa (meta integral):** calcula a taxa com base na meta total da entrega (`meta_json`).

As duas abordagens respondem perguntas diferentes e devem ser lidas em conjunto.

## 2. Consulta completa (script Python, sem credenciais)

Script de referencia com credenciais mascaradas. Preencha `<HOST>`, `<PORT>`, `<DATABASE>`, `<USER>` e `<PASS>` apenas no ambiente local.

```python
"""IND_03.1_run_total.py
Serie temporal do I03 — Taxa de Cumprimento de Metas por Entrega.
"""
import csv
import json
import os
import re
import sys
from datetime import date, datetime

import jpype
import jpype.imports

sys.stdout.reconfigure(encoding="utf-8", errors="replace")

# ── Conexao ─────────────────────────────────────────────────────────────────---
JAR = (
    r"C:\Users\leand\AppData\Roaming\DBeaverData"
    r"\drivers\remote\drivers\jdbc\9\denodo-vdp-jdbcdriver.jar"
)
JVM_DLL = r"C:\Program Files\DBeaver\jre\bin\server\jvm.dll"
JDBC_URL = "jdbc:vdb://<HOST>:<PORT>/<DATABASE>"
USER = "<USER>"
PASS = "<PASS>"

OUT_DIR = os.path.dirname(os.path.abspath(__file__))

SQL_I03 = """
WITH parametros AS (
    SELECT
        CAST('{ini}' AS DATE) AS data_inicio,
        CAST('{fim}' AS DATE) AS data_fim,
        0                     AS incluir_excluidos
),
entregas_base AS (
    SELECT
        u.sigla AS unidade_sigla,
        u.nome  AS unidade_nome,
        pee.id  AS id_entrega,
        COALESCE(
            NULLIF(TRIM(pee.descricao), ''),
            NULLIF(TRIM(pee.descricao_entrega), ''),
            'N.I.'
        ) AS nome_entrega,
        COALESCE(NULLIF(TRIM(pee.descricao_entrega), ''), 'N.I.') AS descricao_entrega,
        pee.progresso_esperado AS progresso_esperado_bruto,
        CASE
            WHEN pee.progresso_esperado > 0 AND pee.progresso_esperado <= 1
            THEN pee.progresso_esperado * 100
            ELSE pee.progresso_esperado
        END AS meta_planejada,
        COALESCE(pee.progresso_realizado, 0) AS meta_executada,
        pee.meta      AS meta_json,
        pee.realizado AS realizado_json
    FROM petrvs_icmbio_planos_entregas pe
    JOIN petrvs_icmbio_planos_entregas_entregas pee
        ON pee.plano_entrega_id = pe.id
    JOIN petrvs_icmbio_unidades u
        ON u.id = pe.unidade_id
    CROSS JOIN parametros p
    WHERE CAST(pee.data_fim AS DATE) BETWEEN p.data_inicio AND p.data_fim
      AND (p.incluir_excluidos = 1 OR pe.deleted_at  IS NULL)
      AND (p.incluir_excluidos = 1 OR pee.deleted_at IS NULL)
      AND pee.progresso_esperado IS NOT NULL
      AND pee.progresso_esperado > 0
),
entregas_com_taxa AS (
    SELECT
        unidade_sigla,
        unidade_nome,
        id_entrega,
        nome_entrega,
        descricao_entrega,
        progresso_esperado_bruto,
        CASE
            WHEN progresso_esperado_bruto > 0 AND progresso_esperado_bruto <= 1
            THEN 'S'
            ELSE 'N'
        END AS anomalia_escala,
        meta_planejada,
        meta_executada,
        ROUND(meta_executada / NULLIF(meta_planejada, 0) * 100, 2) AS taxa_atingimento_perc,
        meta_json,
        realizado_json
    FROM entregas_base
)
SELECT
    unidade_sigla,
    unidade_nome,
    id_entrega,
    nome_entrega,
    descricao_entrega,
    progresso_esperado_bruto,
    anomalia_escala,
    meta_planejada,
    meta_executada,
    taxa_atingimento_perc,
    CASE
        WHEN taxa_atingimento_perc >  100 THEN 'Superexecutada'
        WHEN taxa_atingimento_perc =  100 THEN 'Concluida'
        WHEN taxa_atingimento_perc >= 70  THEN 'Parcialmente cumprida'
        WHEN taxa_atingimento_perc >    0 THEN 'Em andamento'
        ELSE                                   'Nao executada'
    END AS status_entrega,
    meta_json,
    realizado_json
FROM entregas_com_taxa
ORDER BY unidade_sigla, taxa_atingimento_perc DESC
"""


def build_periods():
    """2025 = trimestral | 2026+ = quadrimestral."""
    today = date.today()
    raw = [
        ("T1-2025", "trimestral",    date(2025, 1,  1), date(2025, 3, 31)),
        ("T2-2025", "trimestral",    date(2025, 4,  1), date(2025, 6, 30)),
        ("T3-2025", "trimestral",    date(2025, 7,  1), date(2025, 9, 30)),
        ("T4-2025", "trimestral",    date(2025, 10, 1), date(2025, 12, 31)),
        ("Q1-2026", "quadrimestral", date(2026, 1,  1), date(2026, 4, 30)),
        ("Q2-2026", "quadrimestral", date(2026, 5,  1), date(2026, 8, 31)),
        ("Q3-2026", "quadrimestral", date(2026, 9,  1), date(2026, 12, 31)),
    ]
    periods = []
    for lbl, tipo, p_ini, p_fim in raw:
        if p_ini > today:
            continue
        status = "em_andamento" if p_fim >= today else "encerrado"
        periods.append((lbl, tipo, p_ini, p_fim, status))
    return periods


def start_jvm():
    if not jpype.isJVMStarted():
        jpype.startJVM(JVM_DLL, classpath=[JAR])
        print("JVM iniciada.")


def get_conn():
    DM = jpype.JClass("java.sql.DriverManager")
    return DM.getConnection(JDBC_URL, USER, PASS)


def clean(val):
    if val is None:
        return ""
    return re.sub(r"[\r\n]+", " / ", str(val)).strip()


def classify_status(taxa):
    if taxa is None:
        return "N.D."
    if taxa > 100:
        return "Superexecutada"
    if taxa == 100:
        return "Concluida"
    if taxa >= 70:
        return "Parcialmente cumprida"
    if taxa > 0:
        return "Em andamento"
    return "Nao executada"


def parse_meta_integral(meta_str, realizado_str):
    try:
        if not meta_str or meta_str.strip() in ("null", "NULL", ""):
            return "", "N.D.", "N.D."
        if not realizado_str or realizado_str.strip() in ("null", "NULL", ""):
            return "", "N.D.", "N.D."
        meta = json.loads(meta_str)
        real = json.loads(realizado_str)
        if "quantitativo" in meta:
            meta_val = float(str(meta["quantitativo"]))
            real_val = float(str(real.get("quantitativo", 0)))
            tipo = "quantitativo"
        elif "porcentagem" in meta:
            meta_val = float(str(meta["porcentagem"]))
            real_val = float(str(real.get("porcentagem", 0)))
            tipo = "porcentagem"
        else:
            return "", "N.D.", "N.D."
        if meta_val <= 0:
            return "", "N.D.", tipo
        taxa = round(real_val / meta_val * 100, 2)
        return str(taxa), classify_status(taxa), tipo
    except (ValueError, TypeError, KeyError, json.JSONDecodeError):
        return "", "N.D.", "N.D."


def run_period(conn, ini, fim):
    sql = SQL_I03.format(ini=ini, fim=fim)
    stmt = conn.createStatement()
    rs = stmt.executeQuery(sql)
    meta = rs.getMetaData()
    n = meta.getColumnCount()
    cols = [str(meta.getColumnLabel(i + 1)) for i in range(n)]
    rows = []
    while rs.next():
        rows.append([clean(rs.getObject(i + 1)) for i in range(n)])
    rs.close()
    stmt.close()
    return cols, rows


def save_csv(path, cols, rows):
    with open(path, "w", newline="", encoding="utf-8-sig") as f:
        csv.writer(f, delimiter="|").writerows([cols] + rows)
    print(f"  Salvo: {os.path.basename(path)}")


def main():
    start_jvm()
    conn = get_conn()
    print("Conexao Denodo OK.\n")

    periods = build_periods()
    meta_cols = [
        "ciclo_tipo", "periodo", "periodo_inicio", "periodo_fim",
        "periodo_status", "duracao_dias",
    ]
    extra_cols = ["taxa_meta_integral_perc", "status_meta_integral", "tipo_meta"]

    all_cols = None
    all_rows = []

    for lbl, tipo, p_ini, p_fim, status in periods:
        cols, rows = run_period(conn, p_ini, p_fim)
        if all_cols is None:
            all_cols = meta_cols + cols + extra_cols
        for row in rows:
            taxa_int, status_int, tipo_meta = parse_meta_integral(row[-2], row[-1])
            all_rows.append([
                tipo, lbl, str(p_ini), str(p_fim), status, (p_fim - p_ini).days + 1
            ] + row + [taxa_int, status_int, tipo_meta])

    conn.close()

    stamp = datetime.now().strftime("%Y%m%d_%H%M")
    fname = os.path.join(OUT_DIR, f"IND_03.2_taxa_cumprimento_temporal_{stamp}.csv")
    save_csv(fname, all_cols, all_rows)


if __name__ == "__main__":
    main()
```

## 3. Passos da consulta (explicacao detalhada)

1. **Definir os periodos de analise.**
    O script constrói os periodos (trimestral para 2025, quadrimestral para 2026+). Cada periodo recebe um rótulo e um status que devem ser usados para interpretar se o resultado é final ou provisório.

2. **Selecionar entregas que vencem no periodo (`entregas_base`).**
    O critério principal utiliza `CAST(pee.data_fim AS DATE) BETWEEN p.data_inicio AND p.data_fim`. Essa decisão garante foco nas entregas cujo prazo foi estipulado dentro do recorte, evitando incluir entregas de ciclos paralelos.

3. **Normalizar escala de metas (`anomalia_escala`).**
    Algumas entregas usam escala 0–1 em vez de 0–100; a consulta converte valores entre 0–1 para escala percentual multiplicando por 100 quando aplicavel e marca a linha com `anomalia_escala = 'S'`. Isso previne leituras erradas de desempenho.

4. **Calcular a taxa por entrega (abordagem principal).**
    `taxa_atingimento_perc` e calculada como `meta_executada / meta_planejada * 100` com `NULLIF` para evitar divisao por zero. Entregas com `meta_planejada` invalida são excluidas do calculo.

5. **Calcular a taxa pela meta integral (Python).**
    O script interpreta os campos JSON `meta` e `realizado`, extrai valores `quantitativo` ou `porcentagem` e calcula `taxa_meta_integral_perc`. Esse calculo funciona como segundo termometro para entregas complexas que nao cabem em um unico numero.

6. **Classificar status e exportar.**
    A consulta atribui `status_entrega` (Superexecutada, Concluida, Parcialmente cumprida, Em andamento, Nao executada) e exporta o CSV com colunas extras para ajudar a auditoria (ex: `anomalia_escala`, `meta_json`, `realizado_json`).

### QT-01 — Distribuicao de planos de entrega por ciclo

```sql
SELECT
    EXTRACT(YEAR  FROM CAST(pe.data_inicio AS DATE)) AS ano,
    EXTRACT(MONTH FROM CAST(pe.data_inicio AS DATE)) AS mes_inicio,
    EXTRACT(MONTH FROM CAST(pe.data_fim   AS DATE))  AS mes_fim,
    COUNT(*)                                          AS total_planos,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct_total
FROM planos_entregas pe
WHERE pe.deleted_at IS NULL
  AND CAST(pe.data_inicio AS DATE) >= CAST('2025-01-01' AS DATE)
GROUP BY
    EXTRACT(YEAR  FROM CAST(pe.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pe.data_inicio AS DATE)),
    EXTRACT(MONTH FROM CAST(pe.data_fim   AS DATE))
ORDER BY ano, mes_inicio
```

## 7. Como interpretar o resultado (exemplos reais)

O I03 entrega um nivel de detalhe util para gestores tacticos e tecnicos. Para cada entrega, o gestor deve perguntar:

- A entrega teve meta bem definida (`progresso_esperado` > 0)?
- A escala da meta foi consistente (0-1 vs 0-100)?
- A `taxa_atingimento_perc` e suportada por `taxa_meta_integral_perc` quando disponivel?

Use os exemplos abaixo (CSV: `IND_03.2_taxa_cumprimento_temporal_20260518_2142.csv`) para guiar interpretacoes.

### Exemplo 1 — Entrega superexecutada (sinal de subestimacao)

Uma entrega com `taxa_atingimento_perc = 350.00` indica que a unidade entregou 3,5 vezes o que havia sido planejado. Isso nao e necessariamente positivo: pode sinalizar que a meta foi subestimada. O gestor deve:

- Verificar `meta_json` para entender se a meta integral e comparavel.
- Conferir `tipo_meta` e `taxa_meta_integral_perc` para confirmar a superexecucao em termos absolutos.
- Se confirmado, discutir com a unidade se as metas futuras podem ser recalibradas.

### Exemplo 2 — Entrega com escala inconsistente

Se `anomalia_escala = 'S'`, a meta original estava em 0–1 e foi convertida para 0–100. Nesses casos, compare `progresso_esperado_bruto` e `meta_planejada` antes de tomar decisoes. A conversao pode mascarar pequenas diferencas quando a meta original era fracao.

### Exemplo 3 — Entrega com discordancia entre abordagens

Uma entrega pode apresentar `taxa_atingimento_perc = 85.00` (abordagem ciclo) e `taxa_meta_integral_perc = 40.00` (meta integral). Nessa situacao, a interpretacao gerencial exige analise qualitativa do `meta_json` e do `realizado_json`: possivelmente a porcao do ciclo foi cumprida, mas a meta integral envolve subitens nao executados.

### Exemplos adicionais

### Exemplo 4 — Entrega com execucao parcial na meta integral

Entrega **"RELATORIO ANUAL DE GESTAO DA ACADEBIO ELABORADO"**:

- `taxa_atingimento_perc = 125.00` (criterio do ciclo)
- `taxa_meta_integral_perc = 50.0` (meta integral)
- `status_meta_integral = Em andamento`

Interpretacao: a entrega superou a parcela do ciclo, mas ainda esta pela metade da meta integral. O gestor deve avaliar se a parte executada corresponde aos subprodutos mais relevantes ou se faltam etapas criticas.

### Exemplo 5 — Entrega com taxa alta e meta integral zerada

Entrega **"PLANEJAMENTO ESTRATEGICO ATUALIZADO"**:

- `taxa_atingimento_perc = 100.00`
- `taxa_meta_integral_perc = 0.0`
- `status_meta_integral = Nao executada`

Interpretacao: houve cumprimento no ciclo, mas a meta integral ainda nao foi registrada como executada. Pode indicar que o registro da meta integral e posterior ou que ha discrepancia entre o que foi contabilizado no ciclo e a entrega final.

### Exemplo 6 — Entrega com execucao parcial na meta integral (25%)

Entrega **"BANCO DE DADOS ATUALIZADO"**:

- `taxa_atingimento_perc = 125.00`
- `taxa_meta_integral_perc = 25.0`
- `status_meta_integral = Em andamento`

Interpretacao: a execucao no ciclo foi alta, mas o objetivo total ainda esta no inicio. O gestor deve priorizar a verificacao qualitativa do que foi efetivamente entregue.
